Author profiles are defined in `/content/authors/authors.yaml`. Each post looks up an author by the `author` frontmatter key, so the `id` field must match exactly.

### Example author entry

```yaml
- id: chigusa-asuha
  name: "Chigusa Asuha"
  avatar: "/snap-star.png"
  bio: "Lead Developer & Creative Technologist. Passionate about React, MDX, and building beautiful web experiences."
  twitter: chigusa
  instagram: reirain_exe
  github: snap-star
  website: https://example.com
```

Use the `id` value in blog frontmatter:

```yaml
author: "chigusa-asuha"
```

### Supported author fields

This project supports a wide author schema. You can add any of these optional handles:

```ts
export interface Author {
  id: string
  name: string
  avatar: string
  bio: string
  twitter?: string
  instagram?: string
  linkedin?: string
  facebook?: string
  tiktok?: string
  bluesky?: string
  threads?: string
  reddit?: string
  mastodon?: string
  youtube?: string
  twitch?: string
  medium?: string
  stackoverflow?: string
  devto?: string
  discord?: string
  telegram?: string
  email?: string
  phone?: string
  location?: string
  company?: string
  role?: string
  github?: string
  website?: string
}
```

If you want to display an additional social icon, update `src/components/blog/AuthorCard.tsx` and add the corresponding icon mapping.

### Author card usage

Authors are shown automatically on blog posts when the `author` frontmatter value matches an `id` in `content/authors/authors.yaml`.

If the author is missing or the `id` is incorrect, the post still renders, but author details and social links will not appear.
