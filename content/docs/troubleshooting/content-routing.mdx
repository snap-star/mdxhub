# Content & Routing

---

## Posts Not Appearing on the Blog

### Post does not show up after creating the file

**Cause (most common):** Missing or invalid frontmatter fields. The content store silently drops posts with missing required fields.

**Fix:** Make sure all five required fields are present and correctly typed:

```yaml
---
title: "My Post"         # ✅ string
date: "2026-06-21"       # ✅ ISO date string YYYY-MM-DD
author: "jane-doe"       # ✅ must exactly match an id in authors.yaml
category: "Tutorial"     # ✅ string
tags: ["react", "mdx"]   # ✅ array of strings — NOT a plain string
---
```

<Callout type="danger" title="Tags must be an array">
A common mistake is writing `tags: "react, mdx"` as a plain string. This will cause a runtime type error. It must be a YAML array: `tags: ["react", "mdx"]` or the block form:
```yaml
tags:
  - react
  - mdx
```
</Callout>

### Post is visible during `dev` but missing from the production build

**Cause:** The post has `draft: true` in its frontmatter, which is filtered out by the content store for production.

**Fix:** Remove the `draft` key or set it to `false`:

```yaml
---
draft: false  # ← or remove this line entirely
---
```

---

### Post date is in the future

**Cause:** Posts are sorted by date descending. A future date does not hide a post, but it will appear at the very top of the feed — which may be unexpected.

**Fix:** Use today's date for newly published posts.

## 404 Errors on Doc Pages

### Visiting `/docs/my-page` returns a 404

**Cause:** One of two things:

1. The file path does not match the expected slug pattern.
2. The `section` folder name in the URL does not match the folder in `content/docs/`.

**How slugs are derived:**

```
content/docs/guides/installation.mdx   → /docs/guides/installation
content/docs/introduction/index.mdx   → /docs/introduction
content/docs/api/reference.mdx        → /docs/api/reference
```

**Fix:** Check that the URL you are visiting exactly matches the file path (minus `/content/docs` prefix and `.mdx` extension). Rename the file or folder if needed.

---

### The sidebar shows pages but clicking them gives a 404

**Cause:** The `docs.$section.$slug.tsx` route handler expects exactly two URL segments after `/docs/`. A deeply nested file like `content/docs/a/b/c.mdx` (three levels deep) will not be matched.

**Fix:** Keep docs to a **maximum of two folder levels deep**:

```
✅ content/docs/guides/installation.mdx   → /docs/guides/installation
✅ content/docs/introduction/index.mdx   → /docs/introduction
❌ content/docs/guides/advanced/deep.mdx  → Cannot be routed
```

## Author Not Found

### Author avatar and bio do not appear on the post page

**Cause:** The `author` value in the post's frontmatter does not match any `id` in `content/authors/authors.yaml`.

**Fix:** Open `content/authors/authors.yaml` and verify the `id` field. IDs are **case-sensitive**:

```yaml
# content/authors/authors.yaml
- id: jane-doe      # ← The id you must use in frontmatter
  name: Jane Doe
  ...
```

```yaml
# your-post.mdx frontmatter
author: "jane-doe"  # ← Must be an exact match
```

If the author does not exist yet, add them to `authors.yaml` first before publishing the post.

## Search Not Finding Content

### New post is not appearing in Cmd+K search results

**Cause:** The Fuse.js search index is built at page load time from the content store. If the content store has not finished loading (status is still `'loading'`), the index will be empty.

**Fix:** Wait for the full page load and try the search again. If it consistently fails, check the browser console for content store errors:

```js
// Open browser console and run:
window.__ZUSTAND_STORE__ // Check if posts/docs arrays are populated
```

If both arrays are empty, the glob import likely failed — check the Vite console for errors during startup.

## Category Filter Not Working

### Clicking a category on the blog page shows no posts

**Cause:** The category filter compares strings with strict equality. If a post has `category: "tutorial"` (lowercase) and the filter button shows `"Tutorial"` (capitalized from another post), they won't match.

**Fix:** Standardize your category names across all posts. Use consistent casing throughout:

```yaml
# ✅ Consistent
category: "Tutorial"

# ❌ Inconsistent — creates duplicate filter entries
category: "tutorial"
category: "TUTORIAL"
```
