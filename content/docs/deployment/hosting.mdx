Because this project is built with Vite as a purely static Single Page Application (SPA), deploying it to modern static hosting providers is incredibly simple. When you run the build command, Vite compiles your entire application, including the MDX files, into highly optimized static assets inside the `dist` directory.

Here is how you can deploy your project to the most popular hosting platforms.

---

## Vercel

Vercel is the creator of Next.js and provides first-class support for Vite applications. It requires zero configuration out of the box.

### Quick Deploy (Vercel CLI)

1. Install the Vercel CLI globally:
   ```bash
   npm i -g vercel
   ```
2. Run the deployment command from your project root:
   ```bash
   vercel
   ```
3. Follow the interactive prompts. Vercel will automatically detect that you are using Vite and configure the build settings (`npm run build` and the `dist` output directory) for you.

### Deploy from GitHub (Continuous Integration)

1. Push your code to a GitHub repository.
2. Log into the [Vercel Dashboard](https://vercel.com/dashboard) and click **Add New** > **Project**.
3. Import your GitHub repository.
4. Leave the default settings (Framework Preset: `Vite`, Build Command: `npm run build`, Output Directory: `dist`).
5. Click **Deploy**. Vercel will now automatically rebuild your site every time you push to the `main` branch.

<Callout type="tip">
Vercel automatically configures a reverse proxy to handle client-side routing. You do not need a `vercel.json` file for React Router to work correctly!
</Callout>

## Netlify

Netlify is another excellent option for static site hosting with built-in CI/CD.

### Quick Deploy (Netlify CLI)

1. Install the Netlify CLI:
   ```bash
   npm install netlify-cli -g
   ```
2. Run the deployment command:
   ```bash
   netlify deploy
   ```
3. Follow the prompts. When asked for the publish directory, specify `dist`.
4. To deploy to production, run:
   ```bash
   netlify deploy --prod
   ```

### Deploy from GitHub

1. Push your code to GitHub.
2. Log into [Netlify](https://app.netlify.com/) and click **Add new site** > **Import an existing project**.
3. Connect your GitHub account and select your repository.
4. Set the **Build command** to `npm run build` and the **Publish directory** to `dist`.
5. Click **Deploy site**.

### Handling React Router on Netlify

Because this is a Single Page Application (SPA), Netlify needs to route all traffic to `index.html`. You **must** create a `public/_redirects` file with the following content:

```text
/*    /index.html   200
```

Alternatively, if you prefer using a `netlify.toml` file in your project root:

```toml
[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

## GitHub Pages

Deploying to GitHub Pages is completely free, but it requires a slightly different Vite configuration since your site will likely be hosted on a sub-path (e.g., `https://username.github.io/repo-name/`).

### Step 1: Update Vite Config

In your `vite.config.ts`, you need to set the `base` property to match your GitHub repository name:

```ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  // If your repo is named "my-blog", set this to "/my-blog/"
  base: '/my-blog/', 
  plugins: [react(), /* mdx plugins... */],
})
```

<Callout type="warning">
If you are deploying to a custom domain (e.g., `www.mywebsite.com`) rather than a `.github.io` subdomain, you should leave `base: '/'` as the default!
</Callout>

### Step 2: Set up GitHub Actions

Create a workflow file in your project at `.github/workflows/deploy.yml`:

```yaml
name: Deploy static content to Pages

on:
  push:
    branches: ['main']
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: true

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'
      - name: Install dependencies
        run: npm ci
      - name: Build
        run: npm run build
      - name: Setup Pages
        uses: actions/configure-pages@v4
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          # Upload entire repository
          path: './dist'
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### Step 3: Enable GitHub Pages

1. Go to your repository **Settings** on GitHub.
2. Navigate to the **Pages** menu on the left sidebar.
3. Under **Build and deployment**, change the **Source** dropdown to **GitHub Actions**.
4. Push your `.github/workflows/deploy.yml` file to the `main` branch. GitHub will automatically run the action and deploy your site!
