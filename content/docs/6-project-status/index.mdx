Welcome to the **Project Status** page. Here you'll find an up-to-date log of newly implemented features, project limitations, and known behavior notes for this MDX platform.

## ✨ Newly Implemented Features

This project now includes several content and routing improvements based on the current codebase:

- **Nested content routing**: Blog posts under `content/blog/**` and docs pages under `content/docs/**` both support nested folders. `index.mdx` inside a folder resolves to the parent folder slug.
- **Filename fallback matching**: Blog routes can resolve posts by their file name when the current route uses the plain filename slug.
- **Relative image resolution**: MDX image sources that use relative paths are resolved against the current content folder and the built content asset map.
- **`.md` + `.mdx` discovery**: Both Markdown and MDX content files are discovered automatically with the Vite glob patterns.
- **Explicit post sorting**: Blog posts now support an `order` field for manual sorting before default date-based sorting.
- **Improved author schema**: The author registry supports an extended set of social links and profile fields.
- **Badge component**: Colorful inline pill component (`<Badge variant="tip" icon="check">Done</Badge>`) with 22+ colors, admonition variants, and 40+ icon names — globally available in MDX.
- **Tooltip component**: Accessible hover/focus tooltip (`<Tooltip content="...">text</Tooltip>`) using `@base-ui/react` primitives — globally available in MDX.
- **Image lightbox viewer**: Any image in MDX content is now clickable and opens a full-screen overlay with zoom, rotate, and keyboard navigation.
- **Featured posts**: Set `featured: true` in frontmatter to highlight posts in a Featured widget on the blog homepage.
- **Series grouping**: Use `series` and `seriesOrder` frontmatter fields to group related posts into navigable series with prev/next links and progress indicators.
- **CC License badges**: Set `cc` in frontmatter (e.g., `cc: "CC-BY-4.0"`) to automatically render a Creative Commons license badge in the post footer.
- **Inline YAML array parsing**: Tags can use inline array syntax (`tags: ["react", "mdx"]`) in addition to the indented list format. Both work correctly.
- **Dev watch script**: Run `npm run dev:watch` alongside the dev server to auto-regenerate `content-index.json`, RSS, and sitemap on every content file save — no full rebuild needed.
- **Lazy-loaded Disqus comments**: The Disqus comment section uses a lazy-loading pattern with `IntersectionObserver` — the embed script is only loaded when the user scrolls near the comments, keeping initial page load fast.

## 🚧 Project Limitations

Because this is a static, file-based content platform, there are some inherent tradeoffs:

1. **Client-side search scaling**: The Fuse.js index is built in the browser from loaded content. Very large libraries may increase initial page load memory usage.
2. **Content is Git-based**: All posts and docs are managed as files in the repository. There is no CMS admin panel or database-backed content editor.
3. **No built-in auth or comments**: Comments and authentication need to be added using third-party services like Giscus, Disqus, Supabase, or similar.

## 🐛 Known Behavior Notes

- **Reading time fallback**: When `readingTime` is not provided in frontmatter, the site estimates reading time from the available content metadata. For exact control, set `readingTime` manually.
- **Author lookup**: If `author` in blog frontmatter does not match any `id` in `content/authors/authors.yaml`, the post still renders but author details are omitted.
- **Link updates after rename**: Renaming content files or folders changes the resulting slug. Update links in other pages when you move or rename a post or docs page.
- **Tags parser**: The custom YAML parser in `scripts/generate-content-index.cjs` handles both inline array (`tags: ["a", "b"]`) and indented list (`tags:\n  - a`) syntax. However, the parser runs before the MDX processor — fields that look like arrays but are quoted strings (e.g., `title: "[something]"`) are safely interpreted as strings.
- **content-index.json regeneration**: After adding or modifying content, the content-index, RSS, and sitemap must be regenerated. Use the `dev:watch` script during development, or run `node scripts/generate-content-index.cjs` manually if you see missing posts, tags, or series data.
- **Post fields are flat, not nested**: `PostIndexEntry` uses flat fields (`featured`, `series`, `cc`, etc.) rather than a nested `frontmatter` object. All code accessing post properties should use `post.featured`, not `post.frontmatter.featured`.
