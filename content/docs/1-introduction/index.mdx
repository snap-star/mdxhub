Welcome to **MDXHub** — a production-ready, zero-configuration content platform that combines the power of React 19, MDX, Vite, and TailwindCSS v4 into a stunning and developer-friendly writing experience.

## What is MDXHub?

MDXHub is a Single Page Application that gives you the best of both worlds:

- **Markdown simplicity** — write content naturally without fighting a CMS
- **React power** — embed interactive components, videos, callouts, and charts anywhere in your prose
- **Zero routing config** — drop an `.mdx` or `.md` file into the right folder and it instantly appears, auto-discovered by Vite's `import.meta.glob`
- **Nested routes supported** — blog and docs support nested folders, any of your file `*.md` or `*.mdx`.
- **Relative asset loading** — images inside `content/blog/**` and `content/docs/**` can be referenced with relative paths in MDX

## Core Architecture

This project is built on the latest 2026 web standards:

1. **React 19** — concurrent features, `use()`, and the new compiler
2. **Vite 8** — lightning-fast HMR and optimized production builds
3. **React Router v8** — file-based-style nested SPA routing
4. **TailwindCSS v4** — CSS-first config with OKLCH color spaces
5. **Zustand v5** — minimal, boilerplate-free global state
6. **Shiki v4** — build-time syntax highlighting with full VS Code theme support
7. **Mermaid** — Diagram and charting tools support
8. **Framer Motion 12** — production-grade animations and page transitions

## Directory Structure

```
reactmdx/
├── content/                  ← All your content lives here
│   ├── blog/                 ← Blog posts and nested folders (auto-discovered)
│   ├── docs/                 ← Documentation pages and nested folders (auto-discovered)
│   ├── authors/
│   │   └── authors.yaml      ← Author profiles registry
│   └── about.mdx             ← Rendered at /about
├── src/
│   ├── components/
│   │   ├── mdx/              ← Global MDX components (Callout, VideoEmbed, etc.)
│   │   ├── blog/             ← Blog-specific components
│   │   └── docs/             ← Docs-specific components
│   ├── layouts/              ← RootLayout, BlogLayout, DocsLayout
│   ├── routes/               ← Route components
│   └── store/                ← Zustand state stores
├── public/                   ← Static assets served at root
└── site.config.json          ← GitHub URL and site-wide settings
```
