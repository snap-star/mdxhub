Welcome to **ReactMDX** — a production-ready, zero-configuration content platform that combines the power of React 19, MDX, Vite, and TailwindCSS v4 into a stunning and developer-friendly writing experience.

## What is ReactMDX?

ReactMDX is a Single Page Application that gives you the best of both worlds:

- **Markdown simplicity** — write content naturally without fighting a CMS
- **React power** — embed interactive components, videos, callouts, and charts anywhere in your prose
- **Zero routing config** — drop an `.mdx` file into the right folder and it instantly appears, auto-discovered by Vite's `import.meta.glob`

## Feature Overview

| Feature | Details |
| :--- | :--- |
| ✍️ **MDX authoring** | Full MDX 3 support — Markdown + React components in the same file |
| 🎨 **Syntax Highlighting** | Shiki v4 with Monokai dark / GitHub Light themes |
| 🌙 **Dark Mode** | Discord-inspired palette with brand blue accents |
| 🔍 **Full-text Search** | Fuse.js powered — press `Ctrl+K` / `Cmd+K` anywhere |
| 📐 **Math (KaTeX)** | Inline and block LaTeX via `remark-math` + `rehype-katex` |
| 🎬 **Video Embeds** | YouTube, Shorts, Vimeo — drop a URL and it just works |
| 📱 **Responsive** | Mobile-first layout with collapsible sidebar on docs |
| 🔗 **SEO Ready** | Per-page title, description, Open Graph, and Twitter Cards |
| ⚡ **Page Transitions** | Framer Motion `AnimatePresence` with scroll-to-top on nav |
| 🏷️ **GitHub Badge** | Navbar badge auto-reads version from `package.json` |
| ©️ **CC Licenses** | One-line Creative Commons badge for any post |

## Core Architecture

This project is built on the latest 2026 web standards:

1. **React 19** — concurrent features, `use()`, and the new compiler
2. **Vite 8** — lightning-fast HMR and optimized production builds
3. **React Router v8** — file-based-style nested SPA routing
4. **TailwindCSS v4** — CSS-first config with OKLCH color spaces
5. **Zustand v5** — minimal, boilerplate-free global state
6. **Shiki v4** — build-time syntax highlighting with full VS Code theme support
7. **Framer Motion 12** — production-grade animations and page transitions

<Callout type="tip">
Press `Ctrl+K` (or `Cmd+K` on Mac) anywhere on the site to instantly search across all blog posts and documentation pages!
</Callout>

## Directory Structure

```
reactmdx/
├── content/                  ← All your content lives here
│   ├── blog/                 ← Blog posts (auto-discovered)
│   ├── docs/                 ← Documentation pages (auto-discovered)
│   ├── authors/
│   │   └── authors.yaml      ← Author profiles registry
│   └── about.mdx             ← Rendered at /about
├── src/
│   ├── components/
│   │   ├── mdx/              ← Global MDX components (Callout, VideoEmbed, etc.)
│   │   ├── blog/             ← Blog-specific components
│   │   └── docs/             ← Docs-specific components
│   ├── layouts/              ← RootLayout, BlogLayout, DocsLayout
│   ├── routes/               ← Route components
│   └── store/                ← Zustand state stores
├── public/                   ← Static assets served at root
└── site.config.json          ← GitHub URL and site-wide settings
```
