## Section 1: Getting Started with React 19

### What is React 19?

React 19 represents a paradigm shift in how we build React applications. Released in late 2024, it introduced several game-changing features:

- **React Compiler** — Automatic memoization without `useMemo`/`useCallback`
- **Server Components** — Components that render exclusively on the server
- **Actions** — First-class support for mutations and form submissions
- **Document Metadata API** — Native `<title>`, `<meta>` support in components
- **Asset Loading** — Suspense-integrated `<link>` and `<script>` loading
- **Web Components** — Full support for custom elements
- **Enhanced Hooks** — `use()` hook for reading promises, context, and more

## Section 2: JSX Deep Dive

### What is JSX?

JSX (JavaScript XML) is a syntax extension for JavaScript that looks similar to HTML. React components return JSX, which gets compiled into `React.createElement()` calls.

```jsx
// JSX looks like HTML
function Greeting() {
  return <h1>Hello, World!</h1>;
}

// Compiles to:
function Greeting() {
  return React.createElement('h1', null, 'Hello, World!');
}
```

### JSX Rules and Syntax

#### 1. Return a Single Root Element

```jsx
// Valid — single parent wrapper
function ValidComponent() {
  return (
    <div>
      <h1>Title</h1>
      <p>Description</p>
    </div>
  );

// Valid — React Fragment (no extra DOM node)
function FragmentComponent() {
  return (

// Valid — named Fragment with key

          <dd>{item.definition}</dd>

      ))}
    </React.Fragment>

}
```

#### 2. Close All Tags

```jsx
// Self-closing tags must have the slash
function SelfClosingTags() {
  return (
    <div>
      <img src="photo.jpg" alt="Description" />
      <input type="text" />
      <br />
      <hr />
    </div>

```

#### 3. Use camelCase for Attributes

```jsx
function Attributes() {
  return (
    <div>
      {/* className instead of class */}
      <div className="container">Content</div>

      {/* htmlFor instead of for */}
      <label htmlFor="input">Label</label>

      {/* camelCase event handlers */}
      <button onClick={handleClick}>Click</button>

      {/* camelCase CSS properties in inline styles */}
      <div style={{ backgroundColor: 'blue', fontSize: 16 }}>

      </div>
    </div>

```

#### 4. Curly Braces for JavaScript Expressions

```jsx
function Expressions() {

  const age = 30;
  const isAdmin = true;
  const skills = ['React', 'TypeScript', 'Node.js'];

        {skills.map(skill => (
          <li key={skill}>{skill}</li>

      </ul>

  );
}
```

## Section 3: Components and Props

### Function Components

Components are the building blocks of React applications. In React 19, function components are the standard (class components are legacy).

```jsx
// Simple function component
function Welcome({ name }) {
  return <h1>Hello, {name}!</h1>;
}

// Arrow function (common convention)
const Welcome = ({ name }) => {
  return <h1>Hello, {name}!</h1>;
};

// Implicit return (for simple components)
const Welcome = ({ name }) => <h1>Hello, {name}!</h1>;
```

### Understanding Props

Props (properties) are read-only inputs passed to components.

```jsx
// Parent component passing props
function App() {

function Welcome({ name, age, isAdmin }) {

```

## Section 4: State Management with useState

### Understanding useState

The `useState` hook lets you add state to function components. State causes re-renders when updated.

```jsx
import { useState } from 'react';

function Counter() {
  // useState returns: [currentValue, setterFunction]
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}
```

### State with Different Data Types

```jsx
function UserProfile() {
  // String state
  const [name, setName] = useState('Alice');

  // Number state
  const [age, setAge] = useState(25);

  // Boolean state
  const [isEditing, setIsEditing] = useState(false);

  // Array state
  const [hobbies, setHobbies] = useState(['Reading', 'Coding']);

  // Object state
  const [user, setUser] = useState({
    firstName: 'Alice',
    lastName: 'Johnson',
    email: 'alice@example.com',
  });

  // Updating object state (must spread existing properties)
  const updateEmail = (newEmail) => {
    setUser({ ...user, email: newEmail });
  };

  // Updating array state
  const addHobby = (hobby) => {
    setHobbies([...hobbies, hobby]);
  };

  const removeHobby = (hobbyToRemove) => {
    setHobbies(hobbies.filter(h => h !== hobbyToRemove));
  };

  return (
    <div>
      {isEditing ? (
        <div>
          <input 
            value={name} 
            onChange={e => setName(e.target.value)} 
          />
          <button onClick={() => setIsEditing(false)}>Save</button>
        </div>
      ) : (
        <div>
          <h2>{name}</h2>
          <button onClick={() => setIsEditing(true)}>Edit</button>
        </div>
      )}

      <ul>
        {hobbies.map(hobby => (
          <li key={hobby}>
            {hobby}
            <button onClick={() => removeHobby(hobby)}>Remove</button>
          </li>
        ))}
      </ul>
      <button onClick={() => addHobby('Gaming')}>Add Hobby</button>
    </div>
  );
}
```

### Functional State Updates

When the new state depends on the previous state, use the functional update form:

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  // Bad — relies on stale closure in async scenarios
  const incrementWrong = () => {
    setTimeout(() => {
      setCount(count + 1); // May use outdated count value
    }, 1000);
  };

  // Good — functional update ensures latest state
  const incrementCorrect = () => {
    setTimeout(() => {
      setCount(prevCount => prevCount + 1); // Always uses latest value
    }, 1000);
  };

  // Multiple rapid updates — functional form is essential
  const tripleIncrement = () => {
    setCount(c => c + 1);
    setCount(c => c + 1);
    setCount(c => c + 1);
  };

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={incrementCorrect}>Delayed +1</button>
      <button onClick={tripleIncrement}>+3</button>
    </div>
  );
}
```

### Lazy State Initialization

For expensive initial state calculations, use a function to avoid recomputation on every render:

```jsx
function ExpensiveComponent() {
  // Bad — runs parseExpensiveData() on every render (ignores result after first)
  const [data, setData] = useState(parseExpensiveData(rawData));

  // Good — only runs once on initial render
  const [data, setData] = useState(() => parseExpensiveData(rawData));

  // Good — lazy init with localStorage
  const [theme, setTheme] = useState(() => {
    return localStorage.getItem('theme') || 'light';
  });

  return <div>{/* ... */}</div>;
}
```

### State in Multiple Components

```jsx
// Lifting state up — share state between siblings via parent
function TemperatureConverter() {
  const [celsius, setCelsius] = useState('');

  const handleCelsiusChange = (value) => {
    setCelsius(value);
  };

  const handleFahrenheitChange = (value) => {
    setCelsius(value !== '' ? (parseFloat(value) - 32) * 5 / 9 : '');
  };

  const fahrenheit = celsius !== '' ? parseFloat(celsius) * 9 / 5 + 32 : '';

  return (
    <div>
      <TemperatureInput
        scale="c"
        value={celsius}
        onChange={handleCelsiusChange}
      />
      <TemperatureInput
        scale="f"
        value={fahrenheit}
        onChange={handleFahrenheitChange}
      />
    </div>
  );
}

function TemperatureInput({ scale, value, onChange }) {
  const label = scale === 'c' ? 'Celsius' : 'Fahrenheit';

  return (
    <div>
      <label>Temperature in {label}:</label>
      <input
        type="number"
        value={value}
        onChange={e => onChange(e.target.value)}
      />
    </div>
  );
}
```

## Section 5: Event Handling in React

### Basic Event Handling

React uses synthetic events — cross-browser wrappers around native events:

```jsx
function ClickHandler() {
  // Named function handler (preferred for readability)
  function handleClick() {
    alert('Button clicked!');
  }

  return <button onClick={handleClick}>Click me</button>;

  // Arrow function inline (for simple cases)
  // return <button onClick={() => alert('Clicked!')}>Click me</button>;
}
```

### Event Object and Parameters

```jsx
function FormEvents() {
  const handleChange = (event) => {
    // event.target — the DOM element that triggered the event
    console.log('Value:', event.target.value);
    console.log('Name:', event.target.name);
    console.log('Type:', event.target.type);
  };

  const handleClick = (itemId, event) => {
    // Access both the item and the event
    console.log('Item ID:', itemId);
    console.log('Event type:', event.type);
  };

  return (
    <div>
```

```jsx
function ValidatedForm() {
  const [values, setValues] = useState({ email: '', password: '' });
  const [errors, setErrors] = useState({});
  const [touched, setTouched] = useState({});

  const validate = (values) => {
    const errors = {};
    if (!values.email) {
      errors.email = 'Email is required';
    } else if (!/\S+@\S+\.\S+/.test(values.email)) {
      errors.email = 'Email is invalid';
    }
    if (!values.password) {
      errors.password = 'Password is required';
    } else if (values.password.length < 8) {
      errors.password = 'Must be at least 8 characters';
    }
    return errors;
  };

  const handleChange = (e) => {
    const { name, value } = e.target;
    setValues(prev => ({ ...prev, [name]: value }));
    // Validate on change only if field was touched
    if (touched[name]) {
      setErrors(validate({ ...values, [name]: value }));
    }
  };

  const handleBlur = (e) => {
    const { name } = e.target;
    setTouched(prev => ({ ...prev, [name]: true }));
    setErrors(validate(values));
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    const validationErrors = validate(values);
    setErrors(validationErrors);
    setTouched({ email: true, password: true });

    if (Object.keys(validationErrors).length === 0) {
      console.log('Submit:', values);
    }
  };

  return (
    <form onSubmit={handleSubmit} noValidate>
      <div>
        <input
          name="email"
          type="email"
          value={values.email}
          onChange={handleChange}
          onBlur={handleBlur}
          placeholder="Email"
          className={errors.email ? 'error' : ''}
        />
        {touched.email && errors.email && (
          <span className="error-message">{errors.email}</span>
        )}
      </div>
      <div>
        <input
          name="password"
          type="password"
          value={values.password}
          onChange={handleChange}
          onBlur={handleBlur}
          placeholder="Password"
          className={errors.password ? 'error' : ''}
        />
        {touched.password && errors.password && (
          <span className="error-message">{errors.password}</span>
        )}

}
```

### Understanding useEffect

`useEffect` runs side effects after rendering. It replaces lifecycle methods from class components.

```jsx
import { useState, useEffect } from 'react';

function Timer() {
  const [seconds, setSeconds] = useState(0);

  // Runs after every render (no dependency array)
  useEffect(() => {
    console.log('Component rendered or updated');
  });

  // Runs once after initial render (empty dependency array)
  useEffect(() => {
    console.log('Component mounted');
  }, []);

  // Runs when 'seconds' changes
  useEffect(() => {
    console.log(`Seconds changed to: ${seconds}`);
  }, [seconds]);

  // Cleanup function — runs before re-run or unmount
  useEffect(() => {
    const intervalId = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);

    return () => {
      clearInterval(intervalId); // Cleanup
    };
  }, []);

  return <p>Elapsed: {seconds}s</p>;
}
```

```jsx
function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : initialValue;
  });

  useEffect(() => {

      const result = await fetchFn();
      setData(result);
```

### When to Use useReducer

Use `useReducer` when state logic becomes complex — multiple sub-values, depends on previous state, or has interdependent fields.

```jsx
  INCREMENT: 'INCREMENT',
  DECREMENT: 'DECREMENT',
  RESET: 'RESET',
  SET_VALUE: 'SET_VALUE',

// Reducer function: (state, action) => newState
function counterReducer(state, action) {
  switch (action.type) {
    case ACTIONS.INCREMENT:
```

### Combined Hook Composition

```jsx
// useForm.js — combines multiple patterns
import { useState, useCallback } from 'react';

function useForm(initialValues, validate, onSubmit) {
  const [values, setValues] = useState(initialValues);
  const [errors, setErrors] = useState({});
  const [touched, setTouched] = useState({});
  const [isSubmitting, setIsSubmitting] = useState(false);

  const handleChange = useCallback((e) => {
    const { name, value, type, checked } = e.target;
    const newValue = type === 'checkbox' ? checked : value;

    setValues(prev => ({ ...prev, [name]: newValue }));

    if (touched[name]) {
      const fieldErrors = validate({ ...values, [name]: newValue });
      setErrors(prev => ({ ...prev, [name]: fieldErrors[name] }));
    }
  }, [values, touched, validate]);

  const handleBlur = useCallback((e) => {
    const { name } = e.target;
    setTouched(prev => ({ ...prev, [name]: true }));
    const fieldErrors = validate(values);
    setErrors(prev => ({ ...prev, [name]: fieldErrors[name] }));
  }, [values, validate]);

  const handleSubmit = useCallback(async (e) => {
    e.preventDefault();
    const formErrors = validate(values);
    setErrors(formErrors);
    setTouched(Object.keys(initialValues).reduce((acc, key) => {
      acc[key] = true;
      return acc;
    }, {}));

    if (Object.keys(formErrors).length === 0) {
      setIsSubmitting(true);
      try {
        await onSubmit(values);
      } finally {
        setIsSubmitting(false);
      }
    }
  }, [values, validate, onSubmit, initialValues]);

  const reset = useCallback(() => {
    setValues(initialValues);
    setErrors({});
    setTouched({});
    setIsSubmitting(false);
  }, [initialValues]);

  return {
    values,
    errors,
    touched,
    isSubmitting,
    handleChange,
    handleBlur,
    handleSubmit,
    reset,
    setValue: (name, value) => setValues(prev => ({ ...prev, [name]: value })),
  };
}

      if (!values.password) errors.password = 'Required';

    }
```

```jsx
function UpdateUserForm({ userId }) {
  const queryClient = useQueryClient();

  const mutation = useMutation({
    mutationFn: updateUser,

    // Optimistic update
    onMutate: async ({ id, data }) => {
      // Cancel outgoing refetches
      await queryClient.cancelQueries({ queryKey: queryKeys.user(id) });

      // Snapshot previous value
      const previousUser = queryClient.getQueryData(queryKeys.user(id));

      // Optimistically update
      queryClient.setQueryData(queryKeys.user(id), old => ({ ...old, ...data }));

      return { previousUser };
    },

    // Rollback on error
    onError: (err, variables, context) => {
      queryClient.setQueryData(
        queryKeys.user(variables.id),
        context.previousUser
      );
    },

    // Always refetch after
    onSettled: (data, error, variables) => {
      queryClient.invalidateQueries({ queryKey: queryKeys.user(variables.id) });
    },
  });

  const handleSubmit = (formData) => {
    mutation.mutate({ id: userId, data: formData });
  };

  return (
    <form onSubmit={handleSubmit}>
      <button type="submit" disabled={mutation.isPending}>
        {mutation.isPending ? 'Saving...' : 'Save Changes'}
      </button>
      {mutation.isError && <p>Error: {mutation.error.message}</p>}
```

## Section 16: Error Boundaries

### Understanding Error Boundaries

Error boundaries catch JavaScript errors in their child component tree, log them, and display a fallback UI instead of crashing the entire app.

### Class-based Error Boundary (Required — no hooks equivalent)

```jsx
import { Component } from 'react';

class ErrorBoundary extends Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, error: null, errorInfo: null };
  }

  // Update state when error occurs
  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  // Side effects when error occurs
  componentDidCatch(error, errorInfo) {
    this.setState({ error, errorInfo });

    // Log to error tracking service
    console.error('ErrorBoundary caught:', error, errorInfo);
    // logErrorToService(error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      // Render custom fallback UI
      return this.props.fallback || (
        <div className="error-boundary">
          <h2>Something went wrong.</h2>
          <details>
            <summary>Error details</summary>
            <pre>{this.state.error?.toString()}</pre>
            <pre>{this.state.errorInfo?.componentStack}</pre>
          </details>
          <button onClick={() => this.setState({ hasError: false, error: null })}>
            Try Again
          </button>
        </div>
      );
    }

    return this.props.children;
  }
}

// Usage
function App() {
  return (
    <ErrorBoundary>
      <MyFeature />
    </ErrorBoundary>
  );
}
```

```jsx
import { useRef, useState, useMemo } from 'react';

function VirtualList({ items, itemHeight = 50, containerHeight = 400 }) {
  const [scrollTop, setScrollTop] = useState(0);
  const containerRef = useRef(null);

  // Calculate visible range
  const totalHeight = items.length * itemHeight;
  const startIndex = Math.floor(scrollTop / itemHeight);
  const visibleCount = Math.ceil(containerHeight / itemHeight);

  const visibleItems = useMemo(() =>

      ...item,
      index: startIndex + index,
    })),
    [items, startIndex, endIndex]

      ref={containerRef}

      onScroll={e => setScrollTop(e.currentTarget.scrollTop)}
```

## Section 21: React Compiler and Automatic Memoization

### Understanding the Problem

In React 18 and earlier, preventing unnecessary re-renders required manual optimization:

```jsx
// React 18 — manual optimization boilerplate
import { useMemo, useCallback, memo } from 'react';

const ExpensiveChart = memo(function ExpensiveChart({ data, onSelect }) {
  const processed = useMemo(() => {
    return data.map(d => ({
      ...d,
      normalized: (d.value - d.min) / (d.max - d.min),
    }));
  }, [data]);

  const handleClick = useCallback((item) => {
    onSelect(item.id);
  }, [onSelect]);

  return <svg>{/* ... */}</svg>;
});
```

### React Compiler — The Solution

React Compiler (codenamed "React Forget") automatically memoizes components at compile time:

```bash
# Install the React Compiler
npm install -D babel-plugin-react-compiler

# babel.config.js
module.exports = {
  plugins: [
    ['babel-plugin-react-compiler', {
      // Configuration options
      target: '19', // React version
    }],
  ],
};
```

```jsx
// React 19 with Compiler — NO manual memoization needed!
function ExpensiveChart({ data, onSelect }) {
  // Compiler automatically memoizes this transformation
  const processed = data.map(d => ({
    ...d,
    normalized: (d.value - d.min) / (d.max - d.min),
  }));

  // Compiler automatically memoizes this callback
  const handleClick = (item) => {
    onSelect(item.id);
  };

  return <svg>{/* ... */}</svg>;
}
```

### How the Compiler Works

1. **Static Analysis**: The compiler analyzes your component at build time
2. **Dependency Tracking**: It determines which values depend on which props/state
3. **Automatic Memoization**: It inserts memoization automatically where beneficial
4. **Bailout**: If code is too dynamic, it skips compilation for that component

### Configuration and Rules

## Section 22: Server Components Architecture

### What are Server Components?

React Server Components (RSC) execute exclusively on the server. They can access server resources directly but cannot use browser APIs or React hooks that require client state.


|---------|-----------------|------------------|
| Execution | Server only | Browser |

| Bundle size | Zero KB | Downloaded |
| Hooks (`useState`) | Not allowed | Allowed |

### Advanced Async Hook Patterns

```jsx
// useAsync — comprehensive async state management
import { useState, useEffect, useCallback, useRef } from 'react';

function useAsync(asyncFunction, immediate = true) {
  const [status, setStatus] = useState('idle');
  const [data, setData] = useState(null);
  const [error, setError] = useState(null);
  const isMounted = useRef(true);

  const execute = useCallback(async (...params) => {
    if (!isMounted.current) return;

    setStatus('pending');
    setData(null);
    setError(null);

    try {
      const response = await asyncFunction(...params);
      if (isMounted.current) {
        setData(response);
        setStatus('success');
      }
      return response;
    } catch (error) {
      if (isMounted.current) {
        setError(error);
        setStatus('error');
      }
      throw error;
    }
  }, [asyncFunction]);

  useEffect(() => {
    isMounted.current = true;
    if (immediate) execute();
    return () => { isMounted.current = false; };
  }, [execute, immediate]);

  return { execute, status, data, error, isLoading: status === 'pending' };
}

// Usage
function SearchResults({ query }) {
  const { execute, data: results, isLoading, error } = useAsync(
    useCallback((q) => fetchSearchResults(q), []),
    false // Don't run immediately
  );

  useEffect(() => {
    if (query) execute(query);
  }, [query, execute]);

  if (isLoading) return <Skeleton />;
  if (error) return <ErrorMessage error={error} />;

  return <ResultsList results={results || []} />;
}
```

```jsx
// useEventListener — declarative event handling
function useEventListener(eventName, handler, element = window, options) {
  const savedHandler = useRef(handler);

  useEffect(() => {
    savedHandler.current = handler;
  }, [handler]);

  useEffect(() => {
    const targetElement = element?.current || element;
    if (!targetElement?.addEventListener) return;

    const eventListener = (event) => savedHandler.current(event);
    targetElement.addEventListener(eventName, eventListener, options);

    return () => {
      targetElement.removeEventListener(eventName, eventListener, options);
    };
  }, [eventName, element, options]);
}

}
```

### Understanding Concurrent React

Concurrent React is a set of features that allow React to prepare multiple versions of the UI simultaneously, enabling smoother user experiences.

### useTransition

Mark non-urgent state updates as transitions to keep the UI responsive:

```jsx
import { useState, useTransition } from 'react';

function TabContainer() {
  const [tab, setTab] = useState('home');
  const [isPending, startTransition] = useTransition();

  const selectTab = (nextTab) => {
    // Urgent update — immediate feedback
    // (none needed here, but could update active indicator)

    // Non-urgent update — wrapped in transition
    startTransition(() => {
      setTab(nextTab); // This may be interrupted by more urgent updates
    });
  };

  return (
    <div>
      <div className="tabs">
        {['home', 'profile', 'settings'].map(t => (
          <button
            key={t}
            className={tab === t ? 'active' : ''}
            onClick={() => selectTab(t)}
          >
            {t}
          </button>
        ))}
      </div>

      {/* Show spinner during transition */}
      {isPending && <div className="spinner" />}

      {/* Old tab stays visible until new tab is ready */}
      <div className={`tab-content ${isPending ? 'dimmed' : ''}`}>
        <TabContent tab={tab} />
      </div>
    </div>
  );
}

  // Each tab may have heavy rendering
```

### Suspense for Data Fetching

```jsx
import { Suspense, useState } from 'react';
import { fetchProfileData } from './api';

// Resource wrapper for Suspense integration
function wrapPromise(promise) {
  let status = 'pending';
  let result;

  const suspender = promise.then(
    (data) => {
      status = 'success';
      result = data;
    },
    (error) => {
      status = 'error';
      result = error;
    }
  );

  return {
    read() {
      if (status === 'pending') throw suspender;
      if (status === 'error') throw result;
      return result;
    },
  };
}

// Component that throws promise (Suspense catches it)

  );
}

// Orchestration

    wrapPromise(fetchProfileData(userId))
  );

  return (
    <Suspense fallback={<ProfileSkeleton />}>

// React 19 with use() — much simpler!
function ProfilePageModern({ userPromise }) {
```

## Section 28: React Internals and Reconciliation

### The Fiber Architecture

React's reconciler (Fiber) is a complete rewrite of the original stack reconciler, enabling features like Concurrent Mode and Suspense.

```
// Fiber Tree Structure (simplified)
// Each component = one Fiber node

<FiberNode>
  type: 'div' | MyComponent | null (fragment)
  key: 'unique-key'
  stateNode: DOMNode | ComponentInstance
  child: <first child Fiber>
  sibling: <next sibling Fiber>
  return: <parent Fiber>
  
  // Work-in-progress (double buffering)
  alternate: <current Fiber | work-in-progress Fiber>
  
  // Update queue
  pendingProps: { className: 'active' }
  memoizedProps: { className: '' }
  memoizedState: <hook state>
  
  // Priority
  lanes: <update priority>
</FiberNode>
```

### Reconciliation Process

```jsx
// When state/props change, React follows this process:

// 1. Render Phase (can be interrupted)
//    - Build a new Fiber tree (work-in-progress)
//    - Compare with current tree (diffing algorithm)
//    - Mark changes needed (effects to run)

// 2. Commit Phase (synchronous, cannot be interrupted)
//    - Apply all DOM changes
//    - Run side effects (useEffect, useLayoutEffect)
//    - Update refs

// The diffing algorithm:
// - Same type at same position → update props
// - Different type → unmount old, mount new
// - Different key → unmount old, mount new
// - Keys help identify items that moved

// This is why keys are critical for lists!
function ListExample() {
  // Without keys: React can't track items efficiently
  // items.map(item => <Item data={item} />)

  // With keys: React tracks items by stable ID
  // items.map(item => <Item key={item.id} data={item} />)
}
```

### Understanding Priority Lanes

```jsx
// React assigns priorities to updates:
// - Sync Lane: Urgent (input, click, key press)
// - Default Lane: Regular updates
// - Transition Lane: Low priority (useTransition)
// - Idle Lane: Background work

// React processes work from highest to lowest priority
// If a higher priority update arrives, lower priority work is interrupted

// You can see this in action:
function PriorityDemo() {
  const [urgent, setUrgent] = useState('');
  const [deferred, setDeferred] = useState('');
  const [, startTransition] = useTransition();

  const handleChange = (e) => {
    const value = e.target.value;

    // Sync update — highest priority
    setUrgent(value);

    // Transition — lower priority, can be interrupted
    startTransition(() => {
      // This might be skipped if more urgent updates arrive
      setDeferred(value);
    });
  };

  return <input value={urgent} onChange={handleChange} />;
}
```

```jsx
class EventBus {
  constructor() {
    this.events = new Map();
  }

  on(event, callback) {
    if (!this.events.has(event)) {
      this.events.set(event, new Set());
    }
    this.events.get(event).add(callback);

    return () => this.events.get(event)?.delete(callback);
  }

  emit(event, data) {
    this.events.get(event)?.forEach(cb => cb(data));
  }
}
```

```jsx
// 3. Replace useFormState with useActionState
// Before:
const [state, formAction] = useFormState(action, initialState);

// After:
const [state, formAction, isPending] = useActionState(action, initialState);

// 4. Adopt Server Components (Next.js / framework-specific)
// Move data fetching to Server Components
// Use 'use client' only for interactive components

// 5. Enable React Compiler (optional, gradual)
// npm install -D babel-plugin-react-compiler
// Remove manual useMemo/useCallback where compiler handles it
```

### Common Breaking Changes

| Feature | React 18 | React 19 |
|---------|----------|----------|
| `forwardRef` | Required | `ref` is a regular prop |
| `useFormState` | Available | Renamed to `useActionState` |
| `act()` | Required import from `react-dom/test-utils` | Exported from `react` |
| Suspense | Thrown to nearest boundary | Unified behavior for all cases |
| Stylesheets | Manual loading | Built-in precedence support |

### Hooks Cheat Sheet

| Hook | Purpose | Example |
|------|---------|---------|
| `useState` | Component state | `const [count, setCount] = useState(0)` |
| `useEffect` | Side effects | `useEffect(() => {}, [deps])` |
| `useContext` | Context consumption | `const value = useContext(MyContext)` |
| `useReducer` | Complex state | `const [state, dispatch] = useReducer(reducer, init)` |
| `useRef` | Mutable refs | `const ref = useRef(null)` |
| `useMemo` | Memoize values | `const result = useMemo(() => compute(), [deps])` |
| `useCallback` | Memoize functions | `const fn = useCallback(() => {}, [deps])` |
| `useLayoutEffect` | Sync layout effects | Same API as useEffect, runs synchronously |
| `useId` | Stable IDs | `const id = useId()` |
| `useTransition` | Non-urgent updates | `const [isPending, start] = useTransition()` |
| `useDeferredValue` | Deferred updates | `const deferred = useDeferredValue(value)` |
| `useActionState` | Form actions | `const [state, action, pending] = useActionState(fn)` |
| `useOptimistic` | Optimistic UI | `const [opt, add] = useOptimistic(state, fn)` |
| `use` | Read promises/context | `const data = use(promise)` |
| `useSyncExternalStore` | External stores | `const state = useSyncExternalStore(sub, get)` |

### JSX Conditionals Quick Reference

```jsx
// If/else
{condition ? <A /> : <B />}

// AND (render if true)
{condition && <Component />}

// OR (fallback)
{value || <Fallback />}

// Null coalescing
{value ?? <Default />}

// Optional chaining
{user?.name}

// Ternary with null
{condition ? <Component /> : null}

// IIFE for complex logic
{(() => {
  if (a) return <A />;
  if (b) return <B />;
  return <C />;
})()}
```
