# 02 — Zod v4

## Install

```bash
pnpm add zod
```

- Zod v4 is now exported from the package root (`import { z } from "zod"`).
- Zod 3 still available via `zod/v3` if needed during migration.

## Why Zod

- Define a schema once → get a **runtime validator** + a **static TS type** for free.
- No duplicate `interface` + manual validation logic.

## Basics

```ts
import { z } from "zod";

const UserSchema = z.object({
  id: z.uuid(),
  name: z.string().min(1).max(100),
  email: z.email(),
  age: z.number().int().positive().optional(),
  role: z.enum(["admin", "user", "guest"]),
});

type User = z.infer<typeof UserSchema>; // static type, no duplication

UserSchema.parse(data);       // throws ZodError on failure
UserSchema.safeParse(data);   // { success, data } | { success, error }
```

## Common building blocks

```ts
z.string().min(3).max(20)
z.number().positive()
z.boolean()
z.array(z.string())
z.object({ ... })
z.union([z.string(), z.number()])
z.literal("active")
z.record(z.string(), z.number())     // dictionary
z.tuple([z.string(), z.number()])
z.nullable(z.string())
z.optional(z.string())
z.stringbool()                        // "true"/"1" → boolean (v4 addition)
```

## Transforms & refinements

```ts
const TrimmedName = z.string().transform((s) => s.trim());

const PasswordConfirm = z
  .object({ password: z.string().min(8), confirm: z.string() })
  .refine((data) => data.password === data.confirm, {
    error: "Passwords don't match",
    path: ["confirm"],
  });
```

- `error` replaces the old fragmented `message`/`invalid_type_error` params (v4 unification).

## Coercion (e.g. form/query params are always strings)

```ts
const PageQuery = z.object({
  page: z.coerce.number().int().min(1).default(1),
});
```

## Reusable schema composition

```ts
const BaseUser = z.object({ id: z.uuid(), name: z.string() });
const AdminUser = BaseUser.extend({ permissions: z.array(z.string()) });
const PublicUser = BaseUser.pick({ name: true });
const PartialUser = BaseUser.partial();
```

## v4 highlights to know

- **14x faster** string parsing, 7x array, 6.5x object parsing vs v3.
- **Zod Mini** (`zod/mini`) — tree-shakable, functional API for bundle-sensitive apps (~2KB).
- Built-in `z.toJSONSchema()` for JSON Schema export.
- Drastically fewer TS type instantiations → faster editor/compiler perf in large codebases.
- Check compatibility before using v4 with libraries expecting v3 internals (e.g. some MCP servers).

## Where it plugs into the stack

- **tRPC**: `.input(UserSchema)` validates procedure input automatically (file 03).
- **Forms**: pair with React Hook Form via `@hookform/resolvers/zod`.
- **Env vars**: validate `process.env` at startup to fail fast on misconfiguration.

```ts
const env = z.object({
  DATABASE_URL: z.url(),
  PORT: z.coerce.number().default(3000),
}).parse(process.env);
```
