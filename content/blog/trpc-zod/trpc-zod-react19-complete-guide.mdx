## What This Guide Covers

This guide is your comprehensive journey through **tRPC v11**, **Zod v3.24+**, and **React 19** — three technologies that, when combined, create a developer experience that feels like magic. We progress from absolute fundamentals through production-grade architecture patterns.

### The Stack
| Technology | Role | Why It Matters |
|-----------|------|----------------|
| **tRPC v11** | Type-safe API layer | End-to-end types without code generation |
| **Zod v3.24+** | Schema validation | Runtime safety with inferred TypeScript types |
| **React 19** | UI framework | Server Components, Actions, and Suspense integration |
| **TanStack Query v5** | Data fetching | The engine behind tRPC's React hooks |
| **TypeScript 5.7+** | Type system | The glue that makes everything work |

### The Philosophy: Type Safety as a Feature

Before we write code, understand this: **tRPC doesn't just add types to your API — it eliminates an entire class of bugs and cognitive overhead.** When your frontend and backend share types:

- **No more API documentation drift** — types ARE the documentation
- **No more `as any` to silence TypeScript** — the types flow from your database to your UI
- **Refactoring becomes fearless** — change a field name, and every usage breaks immediately
- **Autocompletion becomes telepathy** — your editor knows what's available before you do

## Glossary

| Term | Definition |
|------|------------|
| **tRPC** | TypeScript RPC — end-to-end typesafe APIs |
| **Zod** | TypeScript-first schema validation with static type inference |
| **Procedure** | A tRPC endpoint (query, mutation, or subscription) |
| **Router** | A collection of procedures, organized hierarchically |
| **Middleware** | Functions that run before/after procedures, can modify context |
| **Context** | Shared data passed to all procedures (user, db, etc.) |
| **Input** | Validated input data for a procedure |
| **Output** | Validated response data from a procedure |
| **Batching** | Combining multiple requests into a single HTTP request |
| **Subscription** | Real-time data flow using WebSocket or SSE |
| **Hydration** | Passing server-fetched data to client-side React |
| **Optimistic Update** | Updating UI before server confirmation for perceived speed |
