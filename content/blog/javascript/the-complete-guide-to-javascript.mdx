# Part 0: Introduction & Philosophy

## What This Guide Covers

This guide is your comprehensive journey through **JavaScript (ECMAScript 2024/2025)** — the programming language that powers the modern web. We progress from writing your first line of code through mastering the language's deepest internals.

### The Learning Path

| Level | Topics | Goal |
|-------|--------|------|
| **Junior** | Syntax, types, control flow, functions, arrays, objects, DOM | Write working programs confidently |
| **Intermediate** | ES6+, async programming, classes, modules, APIs, error handling | Build real-world applications |
| **Expert** | Closures, event loop, memory management, design patterns, performance | Architect production-grade systems |

### JavaScript Today

JavaScript in 2026 is not the same language it was a decade ago:

- **ES2024** brought grouped `async` operations, resizable buffers, and `Promise.withResolvers()`
- **ES2025** introduces new array methods, `Iterator` helpers, and RegExp improvements
- **TypeScript** has become the de facto standard for serious JavaScript projects
- **Node.js**, **Deno**, and **Bun** extend JavaScript far beyond the browser

### How to Read This Guide

# Part 1: Junior Level — Foundations

## Chapter 1: Your First JavaScript

<Badge variant="green">Junior</Badge>

### 1.1 Running JavaScript

JavaScript runs in multiple environments. Here are the three easiest ways to start:

**Option A: Browser Console (Fastest)**

Open any web page, press `F12`, click the **Console** tab, and type:

```javascript
console.log("Hello, World!");
// → Hello, World!
```

**Option B: HTML File**

Create `index.html`:

```html
<!DOCTYPE html>
<html>
<head>
  <title>My First JS</title>
</head>
<body>
  <script>
    console.log("Hello from HTML!");
  </script>
</body>
</html>
```

Or link an external file:

```html
<script src="app.js"></script>
```

**Option C: Node.js (Terminal)**

```bash
# Install Node.js from https://nodejs.org
node -v   # Check version (v22+ recommended)

# Run a file
node app.js

# Or use the REPL (Read-Eval-Print Loop)
node
> console.log("Hello from Node!")
Hello from Node!
> 2 + 2
4
```

### 1.2 Statements and Semicolons

JavaScript is executed as a sequence of **statements**. Each statement is an instruction.

```javascript
// A statement performs an action
console.log("This is a statement");

// Statements are separated by semicolons (mostly optional)
let a = 1; let b = 2;

// JavaScript has Automatic Semicolon Insertion (ASI)
// But ALWAYS use semicolons to avoid subtle bugs
let c = 3  // ← No semicolon — works, but risky
let d = 4  // ← This line could merge with the previous one

// Bad: returns undefined because ASI inserts after 'return'
function bad() {
  return           // ← ASI inserts here!
    { value: 42 }  // ← This becomes unreachable
}

// Good: keep return value on same line
function good() {
  return { value: 42 };
}
```

### 1.3 Comments

```javascript
// Single-line comment

/*
  Multi-line comment.
  Use this for longer explanations.
*/

/**
 * JSDoc comment — used for documentation
 * @param {number} x — the input value
 * @returns {number} — the doubled value
 */
function double(x) {
  return x * 2;
}
```

### 1.4 `console` Methods

```javascript
// Logging
console.log("General output");
console.info("Informational message");
console.warn("Warning!");
console.error("Error occurred!");

// Styled logging
console.log("%cStyled text", "color: blue; font-size: 20px;");

// Tables — great for arrays/objects
const users = [{ name: "Alice", age: 30 }, { name: "Bob", age: 25 }];
console.table(users);

// Grouped logging
console.group("User Details");
  console.log("Name: Alice");
  console.log("Age: 30");
console.groupEnd();

// Timing code
console.time("loop");
for (let i = 0; i < 1000000; i++) {}
console.timeEnd("loop");  // → loop: 2.345ms

// Stack trace
console.trace("Where was this called?");
```

## Chapter 2: Variables and Data Types

<Badge variant="green">Junior</Badge>

### 2.1 Declaring Variables

JavaScript has three ways to declare variables. **Use `const` by default, `let` when reassignment is needed, and never use `var`.**

```javascript
// const — value cannot be reassigned (preferred)
const PI = 3.14159;
PI = 3;  // ❌ TypeError: Assignment to constant variable

// const objects/arrays can have their contents modified
const person = { name: "Alice" };
person.name = "Bob";     // ✓ This works!
person = {};             // ❌ TypeError

// let — value can be reassigned
let count = 0;
count = 1;  // ✓ This works

// var — legacy, function-scoped, avoid it
var old = "don't use this";
```

### 2.2 Naming Rules

```javascript
// Valid names
const userName = "Alice";     // camelCase (preferred in JS)
const _private = "value";     // underscore prefix for private-ish
const $element = null;        // dollar sign (common for DOM)
const user2 = "Bob";          // numbers allowed (not at start)
const MAX_SIZE = 100;         // UPPER_SNAKE_CASE for constants

// Invalid names
const 2user = "no";           // ❌ Cannot start with number
const user-name = "no";       // ❌ Hyphens not allowed
const class = "no";           // ❌ Reserved word

// Reserved words you cannot use as variable names:
// break, case, catch, class, const, continue, debugger, default, delete,
// do, else, export, extends, finally, for, function, if, import, in,
// instanceof, new, return, super, switch, this, throw, try, typeof,
// var, void, while, with, yield, let, static, await, enum
```

### 2.3 Primitive Data Types

JavaScript has **7 primitive types** — immutable, stored by value.

```javascript
// === string ===
const name = "Alice";           // Double quotes
const greeting = 'Hello';       // Single quotes
const template = `Hi, ${name}`; // Template literal (backticks)

// String concatenation
const full = "Hello, " + name;  // "Hello, Alice"
const better = `Hello, ${name}`; // "Hello, Alice" (preferred)

// === number ===
const integer = 42;
const float = 3.14;
const negative = -10;
const scientific = 1.5e3;       // 1500
const hex = 0xff;               // 255 (hexadecimal)
const binary = 0b1010;          // 10 (binary)
const octal = 0o77;             // 63 (octal)

// Special number values
console.log(typeof NaN);        // "number" (Not a Number)
console.log(1 / 0);             // Infinity
console.log(-1 / 0);            // -Infinity

// === bigint === (for very large integers)
const huge = 9007199254740991n;
const bigger = huge + 1n;
// Cannot mix BigInt and Number: 1n + 1 → TypeError

// === boolean ===
const isActive = true;
const isComplete = false;

// === undefined ===
let notAssigned;
console.log(notAssigned);       // undefined
console.log(typeof notAssigned); // "undefined"

// === null ===
const empty = null;
console.log(typeof empty);       // "object" (known bug, won't be fixed)

// === symbol === (unique identifier)
const id1 = Symbol("id");
const id2 = Symbol("id");
console.log(id1 === id2);        // false — every Symbol is unique

// === typeof operator ===
console.log(typeof "hello");     // "string"
console.log(typeof 42);          // "number"
console.log(typeof 42n);         // "bigint"
console.log(typeof true);        // "boolean"
console.log(typeof undefined);   // "undefined"
console.log(typeof Symbol());    // "symbol"
console.log(typeof null);        // "object" (quirk!)
console.log(typeof {});          // "object"
console.log(typeof function(){});// "function"
```

### 2.4 The Object Type

Objects are **reference types** — mutable, stored by reference.

```javascript
// Object literal
const user = {
  name: "Alice",
  age: 30,
  isAdmin: false,
  "favorite color": "blue",     // Quoted key with space
  hobbies: ["reading", "gaming"], // Array as value
  address: {                    // Nested object
    city: "New York",
    zip: "10001"
  }
};

// Accessing properties
console.log(user.name);           // Dot notation: "Alice"
console.log(user["favorite color"]); // Bracket notation: "blue"

// Adding/modifying properties
user.email = "alice@example.com";
user.age = 31;

// Deleting properties
delete user.isAdmin;

// Check if property exists
console.log("name" in user);      // true
console.log(user.hasOwnProperty("name")); // true (ignores inherited)

// Check all keys/values
console.log(Object.keys(user));   // ["name", "age", "favorite color", ...]
console.log(Object.values(user)); // ["Alice", 31, "blue", ...]
console.log(Object.entries(user)); // [["name", "Alice"], ["age", 31], ...]

// Array (special type of object)
const fruits = ["apple", "banana", "cherry"];
console.log(fruits[0]);           // "apple" (zero-indexed)
console.log(fruits.length);       // 3
```

### 2.5 `null` vs `undefined` vs undeclared

```javascript
// undefined — variable exists but has no value
let x;
console.log(x);                   // undefined

// null — explicitly "no value" (intentional absence)
let y = null;
console.log(y);                   // null

// undeclared — variable doesn't exist at all
// console.log(z);                // ❌ ReferenceError: z is not defined

// Best practice: use null for intentional emptiness
const findUser = (id) => {
  if (id === 0) return null;      // Explicitly "not found"
  return { id, name: "Alice" };   // Found a user
};

// Check for null OR undefined (but not false, 0, "")
const result = findUser(0);
if (result == null) {             // Loose equality catches both
  console.log("User not found");
}

// Strict check for only null
if (result === null) {
  console.log("Explicitly null");
}
```

## Chapter 3: Operators

<Badge variant="green">Junior</Badge>

### 3.1 Arithmetic Operators

```javascript
// Basic arithmetic
console.log(5 + 3);     // 8  (addition)
console.log(10 - 4);    // 6  (subtraction)
console.log(6 * 7);     // 42 (multiplication)
console.log(20 / 4);    // 5  (division)
console.log(17 % 5);    // 2  (remainder)
console.log(2 ** 8);    // 256 (exponentiation, ES2016)

// Increment / decrement
let n = 5;
console.log(++n);       // 6 (pre-increment: add 1, then return)
console.log(n++);       // 6 (post-increment: return, then add 1)
console.log(n);         // 7

// String concatenation with +
console.log("Hello " + "World"); // "Hello World"
console.log("Age: " + 25);       // "Age: 25" (number coerced to string)

// Unary plus (converts to number)
console.log(+"42");      // 42
console.log(+true);      // 1

// Unary minus (negates)
console.log(-5);         // -5
console.log(-(-5));      // 5
```

### 3.2 Comparison Operators

```javascript
// === Strict equality (ALWAYS USE THIS)
console.log(5 === 5);    // true
console.log(5 === "5");  // false (different types)
console.log(true === 1); // false

// == Loose equality (AVOID — type coercion causes bugs)
console.log(5 == "5");   // true  (string "5" coerced to number 5)
console.log(true == 1);  // true
console.log(false == 0); // true
console.log(null == undefined); // true
console.log("" == 0);    // true
console.log("" == false);// true

// !== Strict inequality
console.log(5 !== "5");  // true

// >, <, >=, <=
console.log(10 > 5);     // true
console.log("apple" < "banana"); // true (string comparison)

// Comparing objects (by reference, not value)
const a = { x: 1 };
const b = { x: 1 };
const c = a;
console.log(a === b);    // false (different objects in memory)
console.log(a === c);    // true (same reference)
```

### 3.3 Logical Operators

```javascript
// && AND — returns first falsy value, or last truthy
console.log(true && true);   // true
console.log(true && false);  // false
console.log("hello" && 42);  // 42 (both truthy, returns last)
console.log(null && "hi");   // null (first falsy, short-circuits)

// || OR — returns first truthy value, or last falsy
console.log(false || true);  // true
console.log(0 || "fallback"); // "fallback"
console.log("hello" || 42);  // "hello" (first truthy, short-circuits)

// ! NOT — inverts boolean
console.log(!true);          // false
console.log(!false);         // true
console.log(!"hello");       // false (string is truthy)
console.log(!"");            // true  (empty string is falsy)
console.log(!!"hello");      // true  (double NOT coerces to boolean)

// ?? Nullish Coalescing (ES2020)
// Returns right side only if left is null or undefined
const value = null ?? "default";       // "default"
const zero = 0 ?? "default";           // 0 (0 is not nullish!)
const emptyStr = "" ?? "default";      // "" (empty string is not nullish!)

// || vs ?? comparison
console.log(0 || "fallback");    // "fallback" (0 is falsy)
console.log(0 ?? "fallback");    // 0        (0 is not nullish) ✓ Better!
```

### 3.4 Truthy and Falsy Values

```javascript
// Falsy values (coerce to false in boolean context)
// There are only 8 falsy values:
console.log(Boolean(false));       // false
console.log(Boolean(0));           // false
console.log(Boolean(-0));          // false
console.log(Boolean(0n));          // false (BigInt zero)
console.log(Boolean(""));          // false (empty string)
console.log(Boolean(null));        // false
console.log(Boolean(undefined));   // false
console.log(Boolean(NaN));         // false

// Everything else is truthy
console.log(Boolean(true));        // true
console.log(Boolean(1));           // true
console.log(Boolean(-1));          // true
console.log(Boolean("hello"));     // true
console.log(Boolean("false"));     // true! (non-empty string)
console.log(Boolean([]));          // true! (empty array)
console.log(Boolean({}));          // true! (empty object)
console.log(Boolean(function(){}));// true
```

### 3.5 Assignment Operators

```javascript
let x = 10;

x += 5;   // x = x + 5;   → 15
x -= 3;   // x = x - 3;   → 12
x *= 2;   // x = x * 2;   → 24
x /= 4;   // x = x / 4;   → 6
x %= 4;   // x = x % 4;   → 2
x **= 3;  // x = x ** 3;  → 8

// Destructuring assignment
const [a, b] = [1, 2];           // a=1, b=2
const { name, age } = { name: "Alice", age: 30 }; // name="Alice", age=30

// Logical assignment operators (ES2021)
let config = null;
config ??= { theme: "dark" };    // config = config ?? { theme: "dark" }

let count = 0;
count ||= 10;  // count = count || 10 → count becomes 10 (0 is falsy)

count &&= 5;   // count = count && 5 → if count is truthy, set to 5
```

### 3.6 The Ternary Operator

```javascript
// condition ? valueIfTrue : valueIfFalse
const age = 20;
const status = age >= 18 ? "adult" : "minor";

// Nested ternaries (use sparingly — can become unreadable)
const category = age < 13 ? "child" : age < 20 ? "teen" : "adult";

// Equivalent with if/else (more readable for complex cases)
let category;
if (age < 13) {
  category = "child";
} else if (age < 20) {
  category = "teen";
} else {
  category = "adult";
}
```

## Chapter 4: Control Flow

<Badge variant="green">Junior</Badge>

### 4.1 `if` / `else if` / `else`

```javascript
const score = 85;

if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 80) {
  console.log("Grade: B");    // This runs
} else if (score >= 70) {
  console.log("Grade: C");
} else if (score >= 60) {
  console.log("Grade: D");
} else {
  console.log("Grade: F");
}

// One-liner (only for single statements)
if (score > 0) console.log("Positive");
```

### 4.2 `switch` Statement

```javascript
const day = 3;

switch (day) {
  case 1:
    console.log("Monday");
    break;  // Don't forget break! Without it, execution falls through
  case 2:
    console.log("Tuesday");
    break;
  case 3:
    console.log("Wednesday");  // This runs
    break;
  case 4:
    console.log("Thursday");
    break;
  case 5:
    console.log("Friday");
    break;
  default:
    console.log("Weekend");
}

// Multiple cases with same result (fall-through is useful here)
switch (day) {
  case 1:
  case 2:
  case 3:
  case 4:
  case 5:
    console.log("Weekday");
    break;
  case 6:
  case 0:
    console.log("Weekend");
    break;
  default:
    console.log("Invalid day");
}
```

### 4.3 `for` Loop

```javascript
// Standard for loop
for (let i = 0; i < 5; i++) {
  console.log(i);  // 0, 1, 2, 3, 4
}

// Loop through array
const colors = ["red", "green", "blue"];
for (let i = 0; i < colors.length; i++) {
  console.log(`${i}: ${colors[i]}`);
}
// → 0: red
// → 1: green
// → 2: blue

// Multiple variables
for (let i = 0, j = 10; i < j; i++, j--) {
  console.log(i, j);  // 0 10, 1 9, 2 8...
}

// Skip iterations with continue
for (let i = 0; i < 10; i++) {
  if (i % 2 === 0) continue;  // Skip even numbers
  console.log(i);  // 1, 3, 5, 7, 9
}

// Break out of loop early
for (let i = 0; i < 100; i++) {
  if (i === 5) break;
  console.log(i);  // 0, 1, 2, 3, 4
}
```

### 4.4 `while` and `do...while` Loops

```javascript
// while — check condition before each iteration
let count = 0;
while (count < 5) {
  console.log(count);  // 0, 1, 2, 3, 4
  count++;
}

// do...while — runs at least once
let num = 0;
do {
  console.log(num);  // 0
  num++;
} while (num < 0);  // Condition is false, but loop ran once

// Reading until condition met
function findFactorial(n) {
  let result = 1;
  while (n > 1) {
    result *= n;
    n--;
  }
  return result;
}
console.log(findFactorial(5));  // 120
```

### 4.5 `for...of` (Iterable Values)

```javascript
// Arrays
const fruits = ["apple", "banana", "cherry"];
for (const fruit of fruits) {
  console.log(fruit);  // apple, banana, cherry
}

// Strings
for (const char of "Hello") {
  console.log(char);   // H, e, l, l, o
}

// With index using entries()
for (const [index, fruit] of fruits.entries()) {
  console.log(`${index}: ${fruit}`);
}

// Can use break/continue
for (const num of [1, 2, 3, 4, 5]) {
  if (num === 3) break;
  console.log(num);  // 1, 2
}
```

### 4.6 `for...in` (Object Keys)

```javascript
// ⚠️ for...in iterates over KEYS (including inherited ones)
const person = { name: "Alice", age: 30, city: "NYC" };

for (const key in person) {
  console.log(`${key}: ${person[key]}`);
}
// → name: Alice
// → age: 30
// → city: NYC

// Always check hasOwnProperty to avoid inherited properties
for (const key in person) {
  if (person.hasOwnProperty(key)) {
    console.log(key);
  }
}

// 💡 Prefer Object.keys(), Object.values(), or Object.entries()
for (const [key, value] of Object.entries(person)) {
  console.log(`${key}: ${value}`);
}
```

## Chapter 5: Functions

<Badge variant="green">Junior</Badge>

### 5.1 Function Declarations

```javascript
// Function declaration — hoisted (can be called before definition)
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Alice"));  // "Hello, Alice!"

// Function expression — NOT hoisted
const sayHi = function(name) {
  return `Hi, ${name}!`;
};
console.log(sayHi("Bob"));    // "Hi, Bob!"

// Named function expression (useful for stack traces)
const factorial = function fact(n) {
  if (n <= 1) return 1;
  return n * fact(n - 1);  // Can reference itself internally
};
```

### 5.2 Arrow Functions (ES6)

```javascript
// Arrow function syntax
const add = (a, b) => a + b;
console.log(add(2, 3));  // 5

// With block body (multiple statements)
const greet = (name) => {
  const greeting = "Hello";
  return `${greeting}, ${name}!`;
};

// Single parameter — parentheses optional
const double = x => x * 2;

// No parameters — empty parentheses required
const getTimestamp = () => Date.now();

// Returning an object (wrap in parentheses)
const makeUser = (name, age) => ({ name, age });
console.log(makeUser("Alice", 30));  // { name: "Alice", age: 30 }

// Arrow functions don't have their own 'this' — inherits from parent
// (This is their most important characteristic — see Expert Chapter)
```

### 5.3 Default Parameters

```javascript
function greet(name = "Guest", greeting = "Hello") {
  return `${greeting}, ${name}!`;
}

console.log(greet());              // "Hello, Guest!"
console.log(greet("Alice"));        // "Hello, Alice!"
console.log(greet("Alice", "Hi"));  // "Hi, Alice!"

// Default with expression
function createId(prefix = "user") {
  return `${prefix}_${Math.random().toString(36).slice(2)}`;
}

// Previous parameter as default
function greetUser(name, greeting = `Hello, ${name}`) {
  return greeting;
}
console.log(greetUser("Alice"));  // "Hello, Alice"
```

### 5.4 Rest Parameters

```javascript
// Rest parameter collects remaining arguments into an array
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}

console.log(sum(1, 2, 3));       // 6
console.log(sum(10, 20));        // 30
console.log(sum());              // 0

// Rest with named parameters
function createUser(name, age, ...tags) {
  return { name, age, tags };
}
console.log(createUser("Alice", 30, "admin", "verified"));
// → { name: "Alice", age: 30, tags: ["admin", "verified"] }

// Must be the last parameter
// function wrong(a, ...rest, b) {}  // ❌ SyntaxError
```

### 5.5 The `arguments` Object (Legacy)

```javascript
// Old way to access all arguments (avoid in modern code — use rest parameters)
function oldSum() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}

// ⚠️ arguments is NOT a real array (no map, filter, etc.)
// ⚠️ Arrow functions don't have arguments object

// Modern approach (preferred):
const modernSum = (...nums) => nums.reduce((a, b) => a + b, 0);
```

### 5.6 Function Scope

```javascript
function outer() {
  const outerVar = "I'm outer";

  function inner() {
    const innerVar = "I'm inner";
    console.log(outerVar);  // ✓ Can access outer scope
    console.log(innerVar);  // ✓ Can access own scope
  }

  inner();
  // console.log(innerVar); // ❌ ReferenceError — innerVar not accessible here
}

outer();

// Variables are function-scoped (var) or block-scoped (let/const)
function scopeDemo() {
  if (true) {
    var functionScoped = "visible throughout function";
    let blockScoped = "only in this block";
    const alsoBlockScoped = "only in this block";
  }
  console.log(functionScoped);        // ✓ Works
  // console.log(blockScoped);        // ❌ ReferenceError
  // console.log(alsoBlockScoped);    // ❌ ReferenceError
}
```

## Chapter 6: Arrays Deep Dive

<Badge variant="green">Junior</Badge>

### 6.1 Creating and Accessing Arrays

```javascript
// Creating arrays
const empty = [];
const fruits = ["apple", "banana", "cherry"];
const mixed = [1, "two", true, null, { x: 1 }, [2, 3]];
const sparse = [1, , 3];  // Sparse array (hole at index 1)

// Accessing elements
console.log(fruits[0]);       // "apple" (zero-indexed!)
console.log(fruits[1]);       // "banana"
console.log(fruits[fruits.length - 1]);  // "cherry" (last element)
console.log(fruits[10]);      // undefined (no error for out of bounds)

// Modifying arrays
fruits[1] = "blueberry";      // Replace
fruits[3] = "date";           // Add at index
fruits.push("elderberry");    // Add to end
fruits.unshift("apricot");    // Add to beginning

// Array length
console.log(fruits.length);   // 6

// Array.isArray check
console.log(Array.isArray(fruits));  // true
```

```javascript
    if (filter === "completed") return todo.completed;
    return true;
  });

    `;

  });
  
  const activeCount = todos.filter(t => !t.completed).length;

  }
```

### 12.2 Promises

A **Promise** represents a value that may not exist yet but will be resolved at some point.

```javascript
// Creating a Promise
const promise = new Promise((resolve, reject) => {
  // Async operation
  setTimeout(() => {
    const success = true;
    if (success) {
      resolve("Operation completed!");  // Fulfill the promise
    } else {
      reject(new Error("Operation failed!"));  // Reject the promise
    }
  }, 1000);
});

// Consuming a Promise
promise
  .then(result => {
    console.log(result);  // "Operation completed!"
    return result.toUpperCase();
  })
  .then(upper => {
    console.log(upper);   // "OPERATION COMPLETED!"
  })
  .catch(error => {
    console.error(error.message);
  })
  .finally(() => {
    console.log("Cleanup: always runs");
  });

// Promise states:
// 1. PENDING — initial state, neither fulfilled nor rejected
// 2. FULFILLED — operation completed successfully
// 3. REJECTED — operation failed
// Once settled (fulfilled or rejected), a promise never changes state.

// Promisifying callbacks
function delay(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

delay(1000).then(() => console.log("1 second passed"));

// Real-world: fetch API (returns a Promise)
fetch("https://api.example.com/users")
  .then(response => {
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    return response.json();  // Returns another Promise
  })
  .then(data => console.log(data))
  .catch(error => console.error("Fetch error:", error));
```

`async/await` is **syntactic sugar over Promises** — it makes async code look synchronous.

```javascript

  if (!response.ok) throw new Error(`HTTP ${response.status}`);
  return await response.json();
}

    console.log(user.name);
  } catch (error) {

  }

  const res = await fetch(`/api/users/${userId}/posts`);
  return res.json();

getValue().then(v => console.log(v));  // 42

  const user = await fetchUser(1);       // Wait for user
  const posts = await fetchPosts(user.id); // Then fetch posts
  const comments = await fetchComments(posts[0].id); // Then comments
  return comments;
}

// Parallel execution (all at once)
async function parallel() {
  const [users, posts, comments] = await Promise.all([
    fetch("/api/users").then(r => r.json()),
    fetch("/api/posts").then(r => r.json()),
    fetch("/api/comments").then(r => r.json()),
  ]);
  return { users, posts, comments };
}
// Total time: max(user_time, posts_time, comments_time)

// Mixed: start parallel, then sequential
async function mixed() {
  // Start both at the same time
  const userPromise = fetchUser(1);
  const settingsPromise = fetchSettings(1);
  
  // Wait for both
  const [user, settings] = await Promise.all([userPromise, settingsPromise]);
  
  // Then do something that needs both
  const personalized = await personalize(user, settings);
  return personalized;
}

// Looping with async/await
async function processUsers(userIds) {
  const results = [];
  for (const id of userIds) {
    // Sequential: one at a time (polite to the server)
    const user = await fetchUser(id);
    results.push(user);
  }
  return results;
}

// ⚠️ forEach doesn't work with async!
// WRONG: All fetches start simultaneously, results lost
userIds.forEach(async (id) => {
  const user = await fetchUser(id);  // Fire-and-forget!
});

// CORRECT: Use for...of for sequential
// CORRECT: Use Promise.all + map for parallel
const users = await Promise.all(userIds.map(id => fetchUser(id)));
```

```javascript
// api/client.js
class ApiClient {
  constructor(baseURL, defaultOptions = {}) {
    this.baseURL = baseURL.replace(/\/$/, "");  // Remove trailing slash
    this.defaultOptions = defaultOptions;
  }

  async request(endpoint, options = {}) {
    const url = `${this.baseURL}${endpoint}`;
    const config = {
      ...this.defaultOptions,
      ...options,
      headers: {
        "Content-Type": "application/json",
        ...this.defaultOptions.headers,
        ...options.headers,
      },
    };

    if (config.body && typeof config.body === "object") {
      config.body = JSON.stringify(config.body);
    }

    const response = await fetch(url, config);
    return this.handleResponse(response);
  }

  async handleResponse(response) {
    if (!response.ok) {
      const error = await response.json().catch(() => ({}));
      throw new ApiError(response.status, error.message || "Request failed");
    }
    
    if (response.status === 204) return null;  // No content
    return response.json();
  }

  // Convenience methods
  get(endpoint, options) {
    return this.request(endpoint, { ...options, method: "GET" });
  }
  
  post(endpoint, body, options) {
    return this.request(endpoint, { ...options, method: "POST", body });
  }
  
  put(endpoint, body, options) {
    return this.request(endpoint, { ...options, method: "PUT", body });
  }
  
  patch(endpoint, body, options) {
    return this.request(endpoint, { ...options, method: "PATCH", body });
  }
  
  delete(endpoint, options) {
    return this.request(endpoint, { ...options, method: "DELETE" });
  }
}

class ApiError extends Error {
  constructor(status, message) {
    super(message);
    this.name = "ApiError";
    this.status = status;
  }
}

// Usage
const api = new ApiClient("https://api.example.com", {
  headers: { "X-API-Version": "v2" },
});

// With auth interceptor
api.request = new Proxy(api.request, {
  apply(target, thisArg, args) {
    const [endpoint, options = {}] = args;
    options.headers = {
      ...options.headers,
      Authorization: `Bearer ${localStorage.getItem("token")}`,
    };
    return target.call(thisArg, endpoint, options);
  },
});

// Use it!
```

### 19.1 How JavaScript Executes

```javascript
// JavaScript is single-threaded with an event loop.
// Understanding the call stack, task queue, and microtask queue is essential.

function a() { b(); }

});
console.log("4");

// 1. console.log("1") — call stack
// 2. setTimeout callback — goes to Web API, then Task Queue
// 3. Promise .then — goes to Microtask Queue
// 4. console.log("4") — call stack
// 5. Call stack empty → check Microtask Queue → console.log("3")
// 6. Microtask Queue empty → check Task Queue → console.log("2")
```

### 19.2 Event Loop Deep Dive

```javascript
// Microtasks vs Macrotasks (Tasks)

// Macrotasks (Task Queue): setTimeout, setInterval, setImmediate, I/O, UI rendering
// Microtasks: Promise.then/catch/finally, queueMicrotask, MutationObserver

// Rule: ALL microtasks execute before the next macrotask

console.log("start");

setTimeout(() => console.log("timeout 1"), 0);
setTimeout(() => console.log("timeout 2"), 0);

Promise.resolve().then(() => {
  console.log("promise 1");

console.log("end");

// 1. "start", "end" — synchronous
// 2. Promise.then callback — microtask, runs before any timeout
//    Inside it: "promise 1", then registers promise 2 microtask and timeout 3
// 3. queueMicrotask — microtask, runs: "microtask 1"
// 4. Check for more microtasks — promise 2 runs: "promise 2"

// 6. Next timeout: "timeout 2"
```

```javascript
// Defer heavy computation without blocking
function processLargeArray(items, chunkSize = 100) {
  return new Promise((resolve) => {
    let index = 0;
    const results = [];
    
    function processChunk() {
      const chunk = items.slice(index, index + chunkSize);

        results.push(heavyComputation(item));

        setTimeout(processChunk, 0);
      } else {

      }
    }
```

## Chapter 20: Closures & Lexical Scope

### 20.1 How Closures Work

A **closure** is a function that remembers and accesses variables from its outer scope, even after the outer function has returned.

```javascript
// Basic closure
function outer() {
  const secret = "I'm hidden";
  
  function inner() {
    console.log(secret);  // inner() remembers secret from outer()
  }
  
  return inner;
}

const fn = outer();   // outer() has returned, its execution context is gone
fn();                  // But inner() still has access to secret!
// → "I'm hidden"

// The closure captures the VARIABLE, not the value
function createCounter() {
  let count = 0;  // This variable lives in the closure
  
  return {
    increment() { return ++count; },
    decrement() { return --count; },
    get() { return count; },
    reset() { count = 0; },
  };
}

const counter = createCounter();
console.log(counter.increment());  // 1
console.log(counter.increment());  // 2
console.log(counter.get());        // 2
counter.reset();
console.log(counter.get());        // 0

// Each call creates a NEW closure with its own variables
const counter2 = createCounter();
console.log(counter2.get());       // 0 — independent!
```

### 20.2 Practical Closure Patterns

```javascript
// === Module Pattern ===
const DataStore = (function() {
  // Private variables (closure)
  let data = {};
  let listeners = [];
  
  // Public API
  return {
    set(key, value) {
      data[key] = value;
      listeners.forEach(fn => fn(key, value));
    },
    get(key) {
      return data[key];
    },
    subscribe(fn) {
      listeners.push(fn);
      // Return unsubscribe function (another closure!)
      return () => {
        listeners = listeners.filter(l => l !== fn);
      };
    },
    getAll() {
      return { ...data };  // Return copy, not reference
    }
  };
})();

const unsub = DataStore.subscribe((key, value) => {
  console.log(`${key} changed to ${value}`);
});
DataStore.set("theme", "dark");  // Logs: "theme changed to dark"
unsub();  // No more notifications

// === Function Factory ===
function makeMultiplier(factor) {
  return function(number) {
    return number * factor;  // factor is captured from closure
  };
}

const double = makeMultiplier(2);
const triple = makeMultiplier(3);
console.log(double(5));  // 10
console.log(triple(5));  // 15

// === Memoization ===
function memoize(fn) {
  const cache = new Map();  // cache lives in closure
  
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) {
      return cache.get(key);
    }
    
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

const fibonacci = memoize(function(n) {
  if (n < 2) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
});

console.log(fibonacci(50));  // Fast due to memoization!

// === Partial Application ===
function partial(fn, ...presetArgs) {
  return function(...laterArgs) {
    return fn(...presetArgs, ...laterArgs);
  };
}

const greet = (greeting, name) => `${greeting}, ${name}!`;
const sayHello = partial(greet, "Hello");
console.log(sayHello("Alice"));  // "Hello, Alice!"

// === Debounce (performance optimization) ===
function debounce(fn, delay) {
  let timerId;  // Lives in closure
  
  return function(...args) {
    clearTimeout(timerId);
    timerId = setTimeout(() => fn.apply(this, args), delay);
  };
}

const handleResize = debounce(() => {
  console.log("Window resized — recalculating layout...");
}, 250);

window.addEventListener("resize", handleResize);

// === Throttle ===
function throttle(fn, limit) {
  let inThrottle;  // Lives in closure
  
  return function(...args) {
    if (!inThrottle) {
      fn.apply(this, args);
      inThrottle = true;
      setTimeout(() => inThrottle = false, limit);
    }
  };
}

const handleScroll = throttle(() => {
  console.log("Scroll position:", window.scrollY);
}, 100);

window.addEventListener("scroll", handleScroll);
```

### 20.3 Closure Gotchas

```javascript
// Classic closure loop bug
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);  // 3, 3, 3 (not 0, 1, 2!)
}
// Why? 'var' is function-scoped. All closures share the same 'i'.

// Fix 1: Use let (block-scoped)
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);  // 0, 1, 2 ✓
}

// Fix 2: IIFE (immediately invoked function expression)
for (var i = 0; i < 3; i++) {
  (function(capturedI) {
    setTimeout(() => console.log(capturedI), 100);
  })(i);
}

// Another common gotcha: closures in object methods
const handlers = [];
for (let i = 0; i < 3; i++) {
  handlers.push({
    value: i,
    log: () => console.log(handlers[i].value)  // ❌ i is captured, but handlers[i] might not exist
  });
}
// Fix: capture the specific value
for (let i = 0; i < 3; i++) {
  const item = { value: i };
  item.log = () => console.log(item.value);
  handlers.push(item);
}
handlers.forEach(h => h.log());  // 0, 1, 2 ✓
```

```javascript
// === Observer / Pub-Sub ===
class EventEmitter {
  #listeners = new Map();
  
  on(event, callback) {
    if (!this.#listeners.has(event)) {
      this.#listeners.set(event, new Set());
    }
    this.#listeners.get(event).add(callback);
    
    // Return unsubscribe
    return () => this.#listeners.get(event)?.delete(callback);
  }
  
  emit(event, data) {
    this.#listeners.get(event)?.forEach(cb => cb(data));
  }
  
  once(event, callback) {
    const wrapper = (data) => {
      callback(data);
      this.off(event, wrapper);
    };
    this.on(event, wrapper);
  }
  
  off(event, callback) {
```

## Project 2: Reactive State Library (Expert)

```javascript
// reactive.js — A simplified Vue-like reactivity system

// === Effect Tracking ===
let activeEffect = null;
const effectStack = [];

function effect(fn, options = {}) {
  const e = createEffect(fn, options);
  e.run();
  return e;
}

function createEffect(fn, options) {
  const e = {
    run() {
      if (e.dirty) {
        try {
          effectStack.push(e);
          activeEffect = e;
          e.dirty = false;
          return fn();
        } finally {
          effectStack.pop();
          activeEffect = effectStack[effectStack.length - 1] || null;
        }
      }
    },
    stop() {
      // Cleanup subscriptions
      e.deps?.forEach(dep => dep.delete(e));
      e.deps = new Set();
    },
    dirty: true,
    deps: new Set(),
    scheduler: options.scheduler,
  };
  return e;
}

// === Reactive Proxy ===
const targetMap = new WeakMap();

function track(target, key) {
  if (!activeEffect) return;
  
  let depsMap = targetMap.get(target);
  if (!depsMap) {
    depsMap = new Map();
    targetMap.set(target, depsMap);
  }
  
  let dep = depsMap.get(key);
  if (!dep) {
    dep = new Set();
    depsMap.set(key, dep);
  }
  
  dep.add(activeEffect);
  activeEffect.deps.add(dep);
}

function trigger(target, key) {
  const depsMap = targetMap.get(target);
  if (!depsMap) return;
  
  const dep = depsMap.get(key);
  if (dep) {
    dep.forEach(e => {
      if (e.scheduler) {
        e.scheduler(e.run);
      } else {
        e.run();
      }
    });
  }
}

function reactive(target) {
  return new Proxy(target, {
    get(target, key) {
      track(target, key);
      return target[key];
    },
    set(target, key, value) {
      const oldValue = target[key];
      target[key] = value;
      if (oldValue !== value) {
        trigger(target, key);
      }
      return true;
    },
  });
}

function ref(value) {
  const r = {
    get value() {
      track(r, "value");
      return value;
    },
    set value(newValue) {
      if (value !== newValue) {
        value = newValue;
        trigger(r, "value");
      }
    },
  };
  return r;
}

function computed(getter) {
  const c = ref();
  effect(() => {
    c.value = getter();
  }, { scheduler: queueJob });
  return c;
}

// === Batching / Scheduler ===
const queue = new Set();
let isFlushing = false;

function queueJob(job) {
  queue.add(job);
  if (!isFlushing) {
    isFlushing = true;
    Promise.resolve().then(() => {
      queue.forEach(job => job());
      queue.clear();
      isFlushing = false;
    });
  }
}

// === Usage ===
const state = reactive({ count: 0, message: "Hello" });
const doubled = computed(() => state.count * 2);

// Track and re-run when state changes
effect(() => {
  console.log(`Count: ${state.count}, Doubled: ${doubled.value}`);
});

state.count++;  // Effect re-runs automatically!
```

```javascript
// 1. Use epsilon comparison
function floatEqual(a, b, epsilon = 1e-10) {
  return Math.abs(a - b) < epsilon;
}
floatEqual(0.1 + 0.2, 0.3);  // true

// 2. Convert to integers for money
const total = (1999 + 499) / 100;  // $19.99 + $4.99 = $24.98

// 3. Use decimal.js library for financial calculations
```

## 5. `==` vs `===`

```javascript
// NEVER use == (loose equality)
0 == "0"       // true
0 == []        // true
"" == false    // true
null == undefined // true
"1" == true    // true

// ALWAYS use === (strict equality)
0 === "0"      // false ✓
0 === []       // false ✓
null === undefined // false ✓

// Exception: checking null or undefined
if (x == null)  // Acceptable: catches both null and undefined
if (x === null || x === undefined)  // Same but more explicit
```

## 6. Async/Await Mistakes

```javascript
// WRONG: No error handling
const data = await fetchData();  // Unhandled rejection if fetch fails!

// RIGHT: Always use try/catch
async function safeFetch() {
  try {
    return await fetchData();
  } catch (error) {
    console.error("Fetch failed:", error);
    return null;
  }
}

// WRONG: forEach with async
items.forEach(async (item) => {  // Fire-and-forget! Errors lost!
  await process(item);
});

// RIGHT: Use for...of for sequential
for (const item of items) {
  await process(item);
}

// RIGHT: Use Promise.all for parallel
await Promise.all(items.map(item => process(item)));

// WRONG: Implicit Promise return in map
const results = items.map(async (item) => {
  return await fetch(item.id);  // Returns Promise, not result!
});
console.log(results);  // [Promise, Promise, Promise] — not the data!

// RIGHT: Await Promise.all
const results = await Promise.all(items.map(async (item) => {
  return await fetch(item.id);
}));
console.log(results);  // [data1, data2, data3] ✓
```
