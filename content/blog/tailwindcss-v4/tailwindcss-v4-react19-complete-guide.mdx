# Part 0: Introduction & Philosophy

## What This Guide Covers

This guide is your comprehensive journey through **Tailwind CSS v4** and **React 19** — two technologies that redefine how we build modern user interfaces. We progress from absolute fundamentals through production-grade design system architecture.

### The Stack
| Technology | Role | Why It Matters |
|-----------|------|----------------|
| **Tailwind CSS v4** | Utility-first CSS framework | CSS-first configuration, zero-build tooling, modern CSS native |
| **React 19** | UI library | Server Components, Actions, enhanced Suspense |
| **TypeScript 5.7+** | Type safety | Autocomplete for Tailwind classes via IntelliSense |
| **PostCSS / Vite** | Build tooling | Lightning-fast compilation with the Tailwind Vite plugin |

### Tailwind CSS v4: What's New

Tailwind CSS v4 is a **ground-up rewrite** that fundamentally changes how you configure and use the framework:

| Feature | v3 (Old) | v4 (New) |
|---------|----------|----------|
| Configuration | `tailwind.config.js` | CSS-first via `@theme` |
| Imports | `@tailwind base/components/utilities` | `@import "tailwindcss"` |
| Custom utilities | JavaScript plugins | `@utility` in CSS |
| Custom variants | JavaScript API | `@variant` in CSS |
| Container queries | Plugin required | Built-in |
| Browser support | Modern | Leverages latest CSS |

### The Philosophy: CSS-First Configuration

**In v4, your CSS file IS your configuration.** Instead of maintaining a separate JavaScript config file, you define everything directly in your stylesheet. This means:

- **No separate build step** for configuration changes — just CSS
- **Native CSS features** like `@property`, `@layer`, and `@supports`
- **Simpler mental model** — one file for all styling concerns
- **Better IDE support** — standard CSS syntax highlighting everywhere

### 1.2 Configuration: The CSS-First Way

**File: `src/styles.css`** — Your configuration AND styles in one file

```css
/* ============================================
   Tailwind CSS v4 — CSS-First Configuration
   ============================================ */

/* Import Tailwind CSS — replaces @tailwind directives */
@import "tailwindcss";

/* ============================================
   THEME CUSTOMIZATION
   Define your design tokens directly in CSS
   ============================================ */

@theme {
  /* Custom colors */
  --color-brand: #0ea5e9;
  --color-brand-dark: #0284c7;
  --color-brand-light: #7dd3fc;
  
  /* Custom fonts */
  --font-sans: "Inter", ui-sans-serif, system-ui, sans-serif;
  --font-mono: "Fira Code", ui-monospace, monospace;
  
  /* Custom spacing scale */
  --spacing-18: 4.5rem;
  --spacing-88: 22rem;
  
  /* Custom breakpoints */
  --breakpoint-3xl: 120rem;
  
  /* Custom animations */
  --animate-fade-in: fade-in 0.3s ease-out;
  --animate-slide-up: slide-up 0.4s ease-out;
}

/* Define the keyframes for custom animations */
@keyframes fade-in {
  from { opacity: 0; }
  to { opacity: 1; }
}

@keyframes slide-up {
  from { opacity: 0; transform: translateY(10px); }
  to { opacity: 1; transform: translateY(0); }
}

/* ============================================
   CUSTOM UTILITIES
   Define your own utility classes
   ============================================ */

@utility text-balance {
  text-wrap: balance;
}

@utility text-pretty {
  text-wrap: pretty;
}

@utility container-tight {
  margin-inline: auto;
  padding-inline: 1rem;
  max-width: 65ch;
}

/* ============================================
   BASE STYLES (replaces @layer base)
   ============================================ */

body {
  font-family: var(--font-sans);
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

/* React 19: Prevent layout shifts with Suspense boundaries */
[data-suspense-boundary] {
  contain: layout style;
}
```

## Chapter 2: The Utility-First Methodology

<Badge variant="green">Junior</Badge>

### 2.1 What Is Utility-First?

**Utility-first CSS** means building interfaces using small, single-purpose utility classes rather than writing custom CSS. Instead of:

```css
/* Traditional approach: custom CSS */
.alert {
  padding: 1rem;
  background-color: #fee2e2;
  border: 1px solid #fecaca;
  border-radius: 0.5rem;
  color: #991b1b;
}
```

You compose utilities directly in your HTML/JSX:

```tsx
// Utility-first approach: compose in JSX
<div className="rounded-lg border border-red-200 bg-red-50 p-4 text-red-800">
  This is an alert message.
</div>
```

### 2.2 Core Utility Categories

Tailwind v4 organizes utilities into these categories:

#### Layout

```tsx
/* === Container === */
<div className="mx-auto max-w-7xl px-4 sm:px-6 lg:px-8">
  {/* mx-auto: center horizontally */}
  {/* max-w-7xl: max-width 80rem */}
  {/* px-4: padding-x 1rem, sm:px-6: 1.5rem on sm+, lg:px-8: 2rem on lg+ */}
</div>

/* === Display === */
<div className="flex">           {/* display: flex */}
<div className="grid">           {/* display: grid */}
<div className="hidden">         {/* display: none */}
<div className="block">          {/* display: block */}
<div className="inline-flex">    {/* display: inline-flex */}

/* === Position === */
<div className="relative">       {/* position: relative */}
<div className="absolute top-0 left-0"> {/* position: absolute, top: 0, left: 0 */}
<div className="fixed inset-0">  {/* position: fixed, inset: 0 */}
<div className="sticky top-0">   {/* position: sticky, top: 0 */}
```

#### Flexbox & Grid

```tsx
/* === Flexbox === */
<div className="flex items-center justify-between gap-4">
  {/* flex: display flex */}
  {/* items-center: align-items center */}
  {/* justify-between: justify-content space-between */}
  {/* gap-4: gap 1rem */}
  
  <div className="flex-1">       {/* flex: 1 1 0% */}
  <div className="flex-shrink-0"> {/* flex-shrink: 0 */}
  <div className="grow">         {/* flex-grow: 1 */}
</div>

<div className="flex flex-col items-start">
  {/* flex-col: flex-direction column */}
  {/* items-start: align-items flex-start */}
</div>

/* === Grid === */
<div className="grid grid-cols-3 gap-4">
  {/* grid-cols-3: 3 equal columns */}
  
  <div className="col-span-2">   {/* span 2 columns */}
  <div className="row-span-2">   {/* span 2 rows */}
  <div className="col-start-2">  {/* start at column 2 */}
</div>

<div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-6">
  {/* Responsive: 1 col mobile, 2 col tablet, 4 col desktop */}
</div>
```

#### Spacing (Padding & Margin)

```tsx
/* === Padding === */
<div className="p-4">            {/* padding: 1rem (all sides) */}
<div className="px-4 py-2">      {/* padding-x: 1rem, padding-y: 0.5rem */}
<div className="pt-8 pr-4 pb-6 pl-2"> {/* individual sides */}

/* === Margin === */
<div className="m-4">            {/* margin: 1rem */}
<div className="mx-auto">        {/* margin-x: auto (center) */}
<div className="mt-4 mb-2">      {/* margin-top: 1rem, margin-bottom: 0.5rem */}
<div className="-ml-2">          {/* negative margin-left: -0.5rem */}
<div className="space-y-4">      {/* adds vertical spacing between children */}
<div className="space-x-2">      {/* adds horizontal spacing between children */}
```

#### Sizing

```tsx
/* === Width === */
<div className="w-full">         {/* width: 100% */}
<div className="w-64">           {/* width: 16rem (256px) */}
<div className="w-1/2">          {/* width: 50% */}
<div className="w-fit">          {/* width: fit-content */}
<div className="w-screen">       {/* width: 100vw */}
<div className="min-w-0">        {/* min-width: 0 (prevents flex overflow) */}

/* === Height === */
<div className="h-full">         {/* height: 100% */}
<div className="h-32">           {/* height: 8rem (128px) */}
<div className="h-screen">       {/* height: 100vh */}
<div className="min-h-[400px]">  {/* min-height: 400px (arbitrary value) */}
<div className="max-h-96 overflow-y-auto"> {/* max-height + scroll */}
```

#### Typography

```tsx
/* === Font Size === */
<p className="text-sm">          {/* font-size: 0.875rem (14px) */}
<p className="text-base">        {/* font-size: 1rem (16px) */}
<p className="text-lg">          {/* font-size: 1.125rem (18px) */}
<p className="text-xl">          {/* font-size: 1.25rem (20px) */}
<p className="text-2xl">         {/* font-size: 1.5rem (24px) */}

/* === Font Weight === */
<p className="font-light">       {/* font-weight: 300 */}
<p className="font-normal">      {/* font-weight: 400 */}
<p className="font-medium">      {/* font-weight: 500 */}
<p className="font-semibold">    {/* font-weight: 600 */}
<p className="font-bold">        {/* font-weight: 700 */}

/* === Line Height === */
<p className="leading-none">     {/* line-height: 1 */}
<p className="leading-tight">    {/* line-height: 1.25 */}
<p className="leading-snug">     {/* line-height: 1.375 */}

<p className="leading-relaxed">  {/* line-height: 1.625 */}

/* === Text Color === */
<p className="text-gray-900">    {/* dark gray */}

<p className="text-left">        {/* text-align: left */}
<p className="text-center">      {/* text-align: center */}
<p className="text-right">       {/* text-align: right */}

<p className="underline underline-offset-4 decoration-blue-500">
  {/* underlined text with offset and colored underline */}
</p>
```

#### Backgrounds & Borders

```tsx
/* === Background Color === */
<div className="bg-white">       {/* background-color: white */}
<div className="bg-gray-50">     {/* very light gray */}
<div className="bg-gray-900">    {/* very dark gray / near black */}
<div className="bg-blue-600">    {/* blue background */}
<div className="bg-transparent"> {/* transparent */}

/* === Background with opacity (v4: use CSS color syntax) */
<div className="bg-blue-600/10"> {/* blue background at 10% opacity */}
<div className="bg-black/50">    {/* black background at 50% opacity */}
```

#### Effects

```tsx
/* === Opacity === */
<div className="opacity-50">     {/* 50% opacity */}
```

## Chapter 3: Responsive Design

### 3.1 Mobile-First Breakpoints

Tailwind v4 uses a **mobile-first** approach. You design for mobile by default, then add breakpoints for larger screens.

```tsx
/* Breakpoint prefixes (mobile-first):
   (none)  = 0px+      (default / mobile)
   sm:     = 40rem+    (640px)
   md:     = 48rem+    (768px)
   lg:     = 64rem+    (1024px)
   xl:     = 80rem+    (1280px)
   2xl:    = 96rem+    (1536px)
*/

<div className="w-full sm:w-1/2 lg:w-1/3">
  {/* Mobile: full width, Tablet: half, Desktop: third */}
</div>

<h1 className="text-xl sm:text-2xl md:text-3xl lg:text-4xl">
  {/* Responsive font sizes */}
</h1>

<div className="p-4 sm:p-6 md:p-8 lg:p-12">
  {/* Responsive padding */}
</div>
```

### 3.2 Hidden/Visible Patterns

```tsx
{/* Show on desktop, hide on mobile */}
<nav className="hidden md:flex">
  Desktop navigation links
</nav>

{/* Show on mobile, hide on desktop */}
<button className="md:hidden">
  Mobile menu button
</button>

{/* Different layouts per breakpoint */}
<div className="flex flex-col md:flex-row">
  <aside className="w-full md:w-64">Sidebar</aside>
  <main className="flex-1">Main content</main>
</div>
```

### 3.3 Container Queries (Built-in v4!)

Tailwind v4 includes **container queries** out of the box — no plugin needed:

```tsx
/* Mark an element as a container */
<div className="@container">
  {/* Child uses container query: @sm means 24rem container width */}
  <div className="grid grid-cols-1 @sm:grid-cols-2 @lg:grid-cols-3 gap-4">
    <Card />
    <Card />
    <Card />
  </div>
</div>

/* Named containers for precise control */
<div className="@container/card">
  <div className="flex flex-col @sm/card:flex-row">
    <img className="w-full @sm/card:w-1/3" src="..." />
    <div className="flex-1">Content</div>
  </div>
</div>

/* Container size breakpoints:
   @xs  = 20rem  (320px)
   @sm  = 24rem  (384px)
   @md  = 28rem  (448px)
   @lg  = 32rem  (512px)
   @xl  = 36rem  (576px)
   @2xl = 42rem  (672px)
   @3xl = 48rem  (768px)
   @4xl = 56rem  (896px)
   @5xl = 64rem  (1024px)
   @6xl = 72rem  (1152px)
   @7xl = 80rem  (1280px)
*/
```

```typescript
import { type ClassValue, clsx } from 'clsx';
import { twMerge } from 'tailwind-merge';

/**
 * Combines clsx (conditional classes) with tailwind-merge
 * (resolves conflicting Tailwind classes).
 * 
 * Usage: cn('text-red-500', isActive && 'text-blue-500')
 * Result: 'text-blue-500' (tailwind-merge resolves the conflict)
 */
export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

```tsx
import { useState, useEffect } from 'react';
```

### 8.3 React 19: useTransition for UI States

```tsx
'use client';

import { useTransition } from 'react';

export function FilterableList({ items }: { items: Item[] }) {
  const [isPending, startTransition] = useTransition();
  const [filter, setFilter] = useState('');
  const [filteredItems, setFilteredItems] = useState(items);

  const handleFilter = (value: string) => {
    setFilter(value);
    
    // Use transition for non-urgent updates
    startTransition(() => {
      const filtered = items.filter((item) =>
        item.name.toLowerCase().includes(value.toLowerCase())
      );
      setFilteredItems(filtered);
    });
  };

  return (
    <div>
      <input
        type="text"
        value={filter}
        onChange={(e) => handleFilter(e.target.value)}
        className="w-full rounded-lg border border-gray-300 px-4 py-2
                   focus:border-blue-500 focus:ring-2 focus:ring-blue-500"
        placeholder="Filter items..."
      />
      
      {/* Show pending state with reduced opacity */}
      <div className={cn('transition-opacity duration-200', isPending && 'opacity-50')}>
        {filteredItems.map((item) => (
          <div key={item.id} className="py-2 border-b border-gray-100">
            {item.name}
          </div>
        ))}
      </div>
    </div>
  );
}
```

### 14.2 Focus Management

```tsx
// Consistent focus styles
<button className="focus-visible:outline-none focus-visible:ring-2 
                   focus-visible:ring-blue-500 focus-visible:ring-offset-2 
                   focus-visible:ring-offset-white">
  Accessible button

// Focus trap for modals
export function FocusTrap({ children }: { children: React.ReactNode }) {
  const containerRef = useRef<HTMLDivElement>(null);
  
  useEffect(() => {
    const container = containerRef.current;
    if (!container) return;
    
    const focusableElements = container.querySelectorAll(
      'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
    );
    const firstElement = focusableElements[0] as HTMLElement;
    const lastElement = focusableElements[focusableElements.length - 1] as HTMLElement;
    
    firstElement?.focus();
    
    const handleTabKey = (e: KeyboardEvent) => {
      if (e.key !== 'Tab') return;
      
      if (e.shiftKey && document.activeElement === firstElement) {
        e.preventDefault();
        lastElement?.focus();
      } else if (!e.shiftKey && document.activeElement === lastElement) {
        e.preventDefault();
        firstElement?.focus();
      }
    };
    
    container.addEventListener('keydown', handleTabKey);
    return () => container.removeEventListener('keydown', handleTabKey);
  }, []);
  
  return <div ref={containerRef}>{children}</div>;
}
```

# Part 5: Common Pitfalls & Solutions

## 1. Class Conflict Resolution

```tsx
// WRONG: Duplicate/conflicting classes
<div className="p-4 p-6">     {/* p-4 is ignored, but order is unpredictable */}
<div className="px-4 px-6">   {/* px-6 wins (last), but don't rely on this */}

// RIGHT: Use cn() (tailwind-merge)
<div className={cn('p-4', isLarge && 'p-6')}>
  {/* cn resolves conflicts — p-6 wins when isLarge is true */}
</div>
```

## 2. Arbitrary Value Abuse

```tsx
// WRONG: Using arbitrary values for things in the default scale
<div className="w-[100%]">    {/* Use w-full instead */}
<div className="p-[1rem]">    {/* Use p-4 instead (1rem = 4 units) */}
<div className="text-[red]">  {/* Use text-red-500 instead */}

// RIGHT: Use the design system
<div className="w-full p-4 text-red-500">
  {/* Consistent, maintainable, and theme-aware */}
</div>

// Arbitrary values are for ONE-OFF cases:
<div className="w-[420px]">   {/* Custom width not in scale */}
<div className="top-[calc(100%+8px)]"> {/* Calculated position */}
```

## 3. Responsive Design Mistakes

```tsx
// WRONG: Desktop-first approach
<div className="lg:w-1/3 w-full"> {/* Works but harder to maintain */}

// RIGHT: Mobile-first
<div className="w-full lg:w-1/3"> {/* Start mobile, add breakpoints */}

// WRONG: Over-specifying
<div className="text-sm sm:text-sm md:text-sm lg:text-lg">
  {/* Redundant — only specify where it changes */}
</div>

// RIGHT: Minimal specification
<div className="text-sm lg:text-lg">
  {/* text-sm applies to mobile, tablet, desktop */}
  {/* lg:text-lg only overrides at lg+ */}
</div>
```

## 4. Missing `shrink-0` on Icons

```tsx
// WRONG: Icons shrink in flex containers
<div className="flex items-center gap-2">
  <Icon className="h-5 w-5" />  {/* Icon shrinks if text is long! */}
  <span>Long text that might cause issues</span>
</div>

// RIGHT: Prevent icon shrinking
<div className="flex items-center gap-2">
  <Icon className="h-5 w-5 shrink-0" />  {/* Icon keeps its size */}
  <span>Long text that might cause issues</span>
</div>
```

## 5. Server vs Client Component Confusion

```tsx
// WRONG: Using browser APIs in Server Components
export function ServerComponent() {
  const [count, setCount] = useState(0); // ❌ Can't use hooks in RSC!
  return <div>{count}</div>;
}

// RIGHT: Split into Server and Client parts
// ServerComponent.tsx — data fetching, static layout
export async function ServerComponent() {
  const data = await fetchData();
  return (
    <div>
      <h1>{data.title}</h1>
      <ClientCounter initial={data.count} /> {/* Client part */}
    </div>
  );
}

// ClientCounter.tsx — interactive part
'use client';
export function ClientCounter({ initial }: { initial: number }) {
  const [count, setCount] = useState(initial); // ✓ Hooks work here
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}
```

- [ ] Mark non-interactive components as Server Components
- [ ] Use `will-change` sparingly on animated elements
- [ ] Implement `content-visibility: auto` for off-screen sections
- [ ] Use `contain: layout` for performance-isolated components
- [ ] Prefer CSS transitions over JS animations
- [ ] Use `@media (prefers-reduced-motion)` for accessibility

# Glossary

| Term | Definition |
|------|------------|
| **Utility-first** | Building interfaces using small, single-purpose CSS classes |
| **Design token** | Named values (colors, spacing) that define a design system |
| **Breakpoint** | Screen width threshold for responsive design changes |
| **Container query** | Responsive styling based on a container's width, not viewport |
| **Arbitrary value** | Custom value in square brackets: `w-[420px]` |
| **`@theme`** | Tailwind v4 directive for defining design tokens in CSS |
| **`@utility`** | Tailwind v4 directive for creating custom utility classes |
| **`@variant`** | Tailwind v4 directive for registering custom state modifiers |
| **`cn()`** | Utility combining `clsx` and `tailwind-merge` for class merging |
| **CVA** | class-variance-authority — library for component variant management |
| **Server Component** | React component that renders on the server (React 19) |
| **Client Component** | React component that hydrates and runs in the browser |
| **Dark mode** | Interface theme optimized for low-light environments |
| **Semantic color** | Color named by purpose (success, error) not appearance (green, red) |
| **Tree-shaking** | Build optimization that removes unused CSS |
